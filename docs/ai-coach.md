# AI 教练

每日打卡式的对话教练，基于你的真实训练数据、目标赛事和长期记忆给个性化建议。

## 初次配置

不需要自己申请或填写第三方服务密钥。

1. 打开 **AI Coach** tab 直接发消息。发送后任务会保存并在后台继续处理，切到别的 App 不会中断；回到 Ultreia 时会自动同步回复。默认会优先使用本机 Codex runner；如果 Codex runner 不可用、超时或返回失败，会自动回退到 DeepSeek，单纯切到后台不会触发回退。
   模型状态前的小圆点表示 Codex runner 状态：绿色 = 下一次预计调用 Codex，橙色 = 正在检查或短暂失去心跳，红色 = 已确认离线、认证异常或 Codex 暂时不可用。刚打开页面时会先按 Codex 可用显示；短暂心跳延迟或单次状态检查失败不会让顶部在 Codex / DeepSeek 之间闪动，只有连续约 30 秒确认异常后才切换，认证失败会立即提示。即使变红，AI Coach 仍会自动改用 DeepSeek。这个状态会在进入页面时检查一次、页面保持打开时约每 5 秒刷新一次、从后台回到前台时再刷新一次。
2. AI Coach 对话、AI 周复盘、计划提炼 / 补救、Memory 更新和每日推送都会走这条 AI 路线。
3. 你在界面里看到的 AI 输出默认使用中文，包括日常对话、每日教练推送、AI 周复盘、日历建议和赛前简报。教练记忆会在后台同时保存中英文对应事实，用于后续判断，不需要人工审核。
4. 当前是个人使用模式，AI 调用不检查钱包余额，也不会扣钱包；界面不显示钱包入口、充值入口和管理员充值入口。
5. 对话历史会自动保存，刷新和换设备都不会丢。手机端为了保持页面切换和滚动顺滑，聊天窗口默认只挂载最近一小段消息；更早的消息会收在顶部的“加载更早”按钮里，点一下会继续展开历史。对话、日报列表和周报正文都优先响应上下滑动；左右滑动仍用于切换底部页面。
6. 手机端输入框右侧有图片按钮和发送按钮；输入框会和这两个按钮贴齐，输入多行时才向上展开。
7. 输入框右侧的图片按钮可以给教练附图，例如训练计划截图、装备照片或体感相关图片。图片会在发送前压缩，只用于本次 Codex 分析；聊天记录里只保存“本次附图”的文字标记，不保存原图。为了交给本机 Codex，图片会短暂进入 AI 任务队列；任务被 runner 领取、完成、失败或超时后，队列里会清理成只含图片名和类型的元数据。DeepSeek 看不了图片，所以带图请求会强制走 Codex；如果 Codex runner 当前不可用，系统会提示重试或改成文字描述，不会把图片请求静默回退到 DeepSeek。
8. AI Coach 顶部把天气和地点分成两个独立图标：点天气会前往日历并高亮今天的天气，点地点会前往 Settings 并高亮“默认位置”。默认天气位置和常用训练地点统一从 Settings → 通知与自动化 → 默认位置管理，可以打开全屏地图选点、保存多个常用地点；天气和教练上下文都会按默认地点刷新。

## 教练每次能看到什么

每次你发消息，教练都会拿到这些信息：

- **个人资料** —— 年龄、性别、职业、训练年限、伤病史、可用器材，以及（如果填了 Resting HR + Max HR）个人心率区间。
- **教练偏好** —— 三个轴（风格、输出长度、风险提醒强度），可在教练设置里用下拉列表调整。
- **训练偏好** —— 可在 AI Coach 设置里按周一到周日、上午 / 晚上选择你通常希望怎么练，可选项是未设置、休息、公路跑、越野跑、力量训练、HIIT、Massage。保存后会跟随 Aevum 账号同步到其他设备。它会进入日常对话、AI 周复盘、赛前简报和日历建议，是默认排课锚点，不是硬约束；你说某天有事、恢复状态不佳、天气或比赛安排需要变化时，教练可以调整并说明原因。
- **教练记忆（Memory facts）** —— Ultreia 自动判断后保留的训练上下文；Current facts 会进入 AI Coach 和 AI 周复盘，Archived 或已到期事实不会进入。
- **当前时间**和**全部目标赛事**。
- **最近一场目标赛事的比赛日天气**（仅当你给那场赛事填了地点；室内的 Hyrox 不算）—— 临近比赛（约两周内）是真实预报，还早则是该地点该日期的往年同期典型天气。教练据此给比赛日的配速 / 补水 / 装备建议、提前提示高温高湿。只带**最近这一场**，后面几个月的比赛不会一并念出来。
- **精选历史赛事**（每个项目只发最近一场 + 最好成绩那场）。
- **最近 8 周的周训练量趋势** —— 每周一行：跑步类（Road Run / Trail Run / Hiking / Floor Climbing）的总距离、总爬升、训练次数。让教练能看出"这周比上周猛涨"这类周期化 / 伤病风险信号，而不只是孤立的最近 10 条。只统计跑步类训练；没有任何跑步数据的周（以及开头连续的空周）不显示。
- **训练负荷与骤增风险** —— 用「时长 × RPE」计算平滑 ACWR：急性负荷按 7 天 EWMA，慢性基线按 4 周 EWMA。教练据此判断你是不是加得太猛、要不要收一收；大训练会逐日衰减，不会过了 7 天突然整块掉出。攒够约 4 周数据后才准（不够时显示"基线建立中"）。
- **最近几天的晨间状态自评** —— 你在日历里打的睡眠 / 腿感 / 精力，让教练判断今天该顶还是该收。
- **计划依从情况** —— 最近两周你排的计划训练，完成了几次、部分完成几次、漏了几次（按同类型实际训练对照计划目标判定）。漏了的，教练会先问你怎么回事，而不是闷头继续加量。
- **最近 10 条已完成训练**（计划训练不包括在内）—— 含你填的 **RPE（自觉用力）**和**备注**，行尾还会附上当时的天气（如果有）。
- **最近 21 天的当日标记** —— 你在日历上勾的 计划休息 / 按摩 / 拉伸 / 生病。教练排计划、评估负荷时会参考。没勾过就不出现。
- **当前天气**（温度、体感、湿度、天气状况、风速、AQI）—— 实时拉取。
- **未来 7 天的天气预报**（每天一行：最高/最低、体感、湿度、天气状况、风速、AQI）—— 无论你那天有没有训练计划，预报都会发给教练，让它在排新计划时主动按天气挑日子。
- **今天和未来约 21 天的计划训练** —— 你排在日历上的计划（不限于一周内）都会发给教练，**所以你可以直接让它评估计划的可行性**（"这周这样排合不合理""这个 long run 安排得对吗"）。今天的计划会明确标成「只是已安排，不代表已经完成」，避免教练把今晚还没发生的训练当作你已经做完。如果计划选择了保存位置，教练会看到这次训练地点；7 天内会尽量附上该位置的当天预报。被你标成 **主课** 的计划也会一起发给教练，作为本周优先保护的锚点。
- **最近几条教练建议反馈** —— 比如哪些日历建议你确认了、跳过了或保存失败了。教练会把这些当成反馈，避免反复提出你已经跳过的方向。

会修改 Calendar 的建议，在生成和真正执行前都会重新读取训练、赛事、晨间状态、Memory facts 和最近建议记录。只要其中一类数据没刷新成功，App 就会暂停这次生成 / 执行并提示重试；普通聊天仍可继续，但会沿用未刷新成功部分的本地数据。App 从后台回到前台一段时间后，也会静默刷新这些上下文。
- **本次教练重点** —— 系统会按你当下的情况，临时给教练加一两条提醒：临近目标赛时提示当前处于堆量 / 巅峰 / 减量哪个阶段；比赛日高温高湿时提醒给热适应建议；负荷骤增时提醒别再硬加；最近漏了计划时提醒先问原因。没触发就不加。
- **完整对话历史**。

> 教练不会主动获取你浏览器之外的任何数据。你给它什么，它就用什么。

### 关于训练数据的新鲜度

每次发消息**之前**都会重新从数据库拉一次最新训练记录，所以无论你是刚在同一个窗口里加的训练，还是在另一台设备上录的训练，下一条 Coach 回复都能看到。

加完一条训练立刻问 "我刚跑完帮我看看"，Coach 应该直接引用那条的距离/时长/心率 —— 如果它说"我看不到你最新的数据"或者"系统还没同步"，那是 LLM 在偷懒**幻觉**，把它当作错答处理，直接反问 "5/24 的数据应该有，请再看一下"，它就会承认。

手动添加或导入活动后，应用会弹出一个确认框。点 **发送给 AI 教练** 后，系统会自动把这次活动发到 AI Coach，并打开 AI Coach 等回复；你不需要再跳过去手动输入或发送。点取消则不会调用 AI。导入活动时，会先等活动保存成功，再弹出发送给教练的确认框，避免教练看不到刚导入的数据。如果你在导入弹窗里写了主观感受，聊天里会显示这句自然备注；后台仍会把新增活动、当前计划、训练负荷、计划依从和近期教练建议一起发给 Coach。教练会同时点评这次训练，并对下一次相关计划给出一个明确口径：保留 / 降量 / 取消 / 暂不判断。教练回复完成后，应用会再用 AI 把你的主观感受压缩成短自评，并把教练回复压缩成短教练点评；如果这一步失败，自评会用本地简短兜底，避免备注丢失。之后点开训练时，会看到「自评 / Self review」和「教练点评 / Coach review」上下竖排展示，每条最多两行。

为了避免刚点评完一条新训练、主动调整又立刻给出另一套口径，导入后主动发送给教练点评时，应用会把当下的计划偏差 / 恢复负荷守门 / 综合调整签名记为“已处理”，并让自动生成的主动调整提醒暂时安静。同一批状态不会马上再次弹出另一张建议卡。

### 你在对话里说的话，优先于日历计划

如果你直接告诉 Coach 一个当下的变更 —— 比如 "我今晚要训练"、"这次长距离我不跑了"、"周六那次挪到周日" —— 它会**以你刚说的为准**，而不是继续按日历里那条旧计划安排。日历计划只是默认值，不是约束。所以想让它换个思路，最快的方式就是直接在对话里说清楚。

## Memory facts（教练记忆）

教练记忆以事实卡片保存，例如：

- 「轻松日必须真的轻松，不然跟腱会犯」
- 「HYROX 只能作为辅助目标，不应取代越野训练主线」
- 「不喜欢被人在恢复跑里提速度」

在 AI Coach 的设置里打开 **Memory**，可以看到 **Current** 和 **Archived** 两类事实，并用 **全部 / 伤病健康 / 目标比赛 / 训练偏好 / 教练风格 / 长期模式** 这些 chip 筛选。五个分类保持固定；展开分类后仍可查看、归档、恢复或删除底层事实，用来纠正明显错误。AI Coach 设置里的 **训练偏好** 是更结构化的每周排课节奏，两者会一起进入教练上下文。

- **Current** —— 当前启用的长期事实和仍在有效期内的短期事实，会进入 AI Coach 和 AI 周复盘。
- **Archived** —— 已失效或已归档事实，仍可查看、恢复或删除，但不会进入教练判断。删除前会再次确认，删除后无法恢复。

**自动记忆管理**默认开启。系统每晚最多整理一次刚过去本地日期里的新对话：没有值得保留的内容时不会写入；有价值的内容会直接新增或合并，不再生成审核卡、顶部提醒或收件箱待处理项。

判断标准如下：

- **长期保留**：稳定的伤病 / 健康约束、目标策略边界、训练偏好、希望教练采用的沟通方式，以及反复出现、会影响未来训练判断的模式。
- **短期保留**：暂时的恢复限制、近期日程 / 装备限制或其它会影响不止一次训练、但预计会结束的上下文。短期事实必须有有效期，到期后自动退出 Current；Archived 中保留 30 天恢复窗口，之后永久清理。
- **直接忽略**：单次训练细节、当天的天气、普通鼓励、一次性建议、短暂情绪、没有证据的推断、已存在内容和只换一两个字的改写。
- **更新或归档**：只有含义发生实质变化才更新；当前只有你明确纠正或撤回时才会自动归档长期事实。某条长期事实当天没被提到，不是归档理由。

训练记录、晨间状态、赛事日期和日历计划本身是各页面的原始数据，不会被复制成记忆，也不会被记忆清理改写。Memory 只保存会影响后续教练判断的派生上下文。

目标赛事里的日期、A / B / C 优先级、距离和爬升继续以 Races 页面为准。如果聊天里的旧说法与当前赛事设置冲突，自动管理优先相信 Races；Memory 最多保留“HYROX 不应取代越野主线”这类跨赛事的长期策略边界。

自动管理的结果随 Aevum 账号跨设备同步。旧的整段文本 Memory 已退出教练判断，AI Coach 和 AI 周复盘只读取 Current Memory facts。

## 收件箱与主动复盘

AI Coach 页顶部的信封是应用内收件箱；旁边的齿轮会打开教练设置。常用入口直接显示 **当前焦点 / 教练偏好 / 训练偏好 / Memory / 最近教练建议**，Prompt 预览、日历建议按钮和清空对话收在 **高级** 里。**当前焦点** 是一个只读汇总，会把长期焦点和短期焦点分开：长期焦点通常是下一场目标赛，短期焦点通常是近期要执行的计划、赛前窗口或当前训练记录重点；下面还会显示正在观察的风险信号，以及可以确认的下一步动作。要改变焦点，需要改它的来源：长期焦点去 Races 调整目标赛，短期焦点去 Calendar 调整未来计划或训练完成状态。每日教练打卡和 AI 周复盘完成提醒会保存在收件箱里；自动记忆管理不会发送待审核提醒。

打开收件箱后是全屏页面，上方有 **每日推送 / 周报 / 待处理** 三个 tab；如果有未读消息，会优先进入第一个有未读的 tab，并在对应 tab 上显示未读数字。每日推送 tab 只放每日教练打卡；周报 tab 放 AI 周复盘；待处理 tab 放仍待确认、保存中或保存失败的日历 / 简报建议，以及其他提醒。已经保存、跳过或取消的建议只在 **最近教练建议** 查看，不会同时堆在收件箱里。待处理底部的清空只删除普通提醒，不会删除建议记录。

每日教练推送在收件箱底部的 **每日推送设置** 里设置时间；标题右侧的 `!` 会显示推送说明。它只在安卓 App 上真正弹系统通知，推送正文默认中文，但运动术语和数字会尽量保留跑者习惯写法，例如 **HYROX**、**8月15日**、**6.2km**。开启后，安卓通知栏会出现一个低优先级、静音的 Ultreia 常驻通知，用来提高系统保留推送连接的概率；通知内容会显示本月的 Sessions / Time 和 Distance / Ascent，并拆成两行对齐显示。这里的 Distance 和 Training 顶部本月 Distance 口径一致，只统计 Road Run / Trail Run / Hiking / Floor Climbing 的距离；HIIT / Strength 即使导入文件里带了距离，也不会算进通知栏 Distance。如果手机系统的电池限制很严格，仍然建议把 Ultreia 加进电池/后台运行白名单。

周报 tab 按日期周期逐行显示，例如 **6.29-7.5**、**6.22-6.28**，时间从近到远排列。周一至周六不会提前显示还没结束的当前周；到周日才新增当前周这一行，即使报告还没生成也可以点进去手动 **分析**。上周和更早的历史周报会继续保留。周报生成完成但正文还没打开时，AI Coach 的信封、周报 tab 和对应日期行会显示未读提醒；真正打开报告正文后才会清除。周报会读取对应时间范围内的完成训练、计划、晨间状态、目标赛事、训练偏好和 Current Memory facts，默认生成中文报告。报告会逐条点评每一次训练，指出本周风险和恢复重点，并给出接下来 7 天的训练安排。

点 **分析** 后，这次请求会保存为后台任务；可以离开周报页、切到别的 App，或暂时关闭 Ultreia，生成结果都会保存到账号。回到前台后，App 会自动同步任务状态和完成结果。生成期间，底部 AI Coach tab、AI Coach 顶部信封、收件箱周报 tab 和对应日期行都会显示运行中的小动画；App 仍被系统允许在后台运行且通知权限正常时，完成后会弹系统通知。回到 AI Coach → 收件箱 → 周报，点对应日期就能看到新报告。周报 tab 底部固定保留 **清空** 和 **周报设置**。进入周报设置后，可以开启自动生成，用滚轮选择星期几和具体时间；标题右侧的 `!` 会显示自动生成说明。服务器会按手机保存的本地时区，在所选时间每周自动生成一次周报，即使 Ultreia 没有打开也会运行。完成后会写入账号里的周报页面，并发送系统通知 / 收件箱提醒。

“周日导入后补漏”也可以在这里开关。如果服务器已经生成过本周周报、之后你又在周日导入了新活动，App 会询问是否立即重新分析，避免周报漏掉刚导入的训练；如果预定生成时间已过但周报仍未生成，也会提供手动分析兜底。预定时间还没到时不会重复打扰。

周报标题右侧有小号的 **分析** 和 **导入计划** 按钮。点击 **导入计划** 会把报告里的具体安排整理成日历建议，你仍然可以逐条修改 / 取消 / 确认后再写入 Calendar；确认前不会自动改日历。分析或提炼正在运行时，按钮会变成 **停止**，点一下会停止当前等待并忽略后续返回结果。

如果周报里有需要你补充的问题，可以在报告底部输入回答并发送到 AI Coach。App 会带上这份周报和你的补充，跳到 AI Coach 继续对话；这个动作不会自动改日历或 Memory。

## 长对话软提示

AI Coach 顶部只保留运行状态、当前模型、天气、地点、收件箱和设置这些高频信息。上下文用量放在 Settings → 高级 → Prompt 预览里；平时不占首屏，接近安全上限时才会在对话上方出现软提示。

当估算上下文接近安全上限时，对话上方会出一条琥珀色软提示，建议你把长期要点固化进 Memory facts；如果教练开始丢早前上下文，再清空对话。提示里直接有**"清空聊天"**按钮，点它清空前会再提醒你先更新一下记忆。

当下一次发送预计会把上下文推到约 **180K** 以上时，AI Coach 会先自动压缩较早的对话：界面会短暂显示“正在压缩上下文”，这期间按钮会被锁住；压缩完成后，旧对话会变成一条可见摘要，并保留最近几轮原文，再继续发送你刚输入的内容。这个压缩摘要只用于维持当前对话上下文，不等同于 Memory facts，也不会自动写入长期记忆。

点提示上的按钮直接跳到 Memory 区域。自动更新 Memory facts 也会调用一次 AI，但不需要余额或充值。

## 把建议导入到日历

只有回复里同时出现明确日期 / 星期和可执行训练安排时，下面才显示 **看看日历建议**。纯分析、复盘、补水睡眠等泛建议不会显示按钮。首次整理会调用一次 AI；同一条回复已经整理过时，按钮会变成 **查看上次整理**，再次点击会直接打开上次结果。

如果最近两周出现连续漏练 / 部分完成，或近期负荷、RPE、晨间状态、训练备注、生病标记显示较明确的恢复风险，AI Coach 对话上方会出现一条主动建议状态条。它会自动生成一份日历建议，不需要你先点“生成”。生成完成后点 **查看建议** 就能打开；确认前仍然不会改动日历，你可以逐条修改、取消或跳过。主动自动调整只处理明天及之后的计划，避免当天训练已经完成后，又回头修改今天那条计划。如果只是漏掉一次普通训练，系统不会立刻自动弹出建议；你仍然可以在 AI Coach 设置里手动点 **一键综合调整**。如果你刚把新增活动发给教练点评过，同一批计划偏差或恢复风险会被视为已处理，避免短时间内重复生成相反建议。

如果计划偏差和恢复风险同时出现，AI Coach 会合并成一份综合建议，而不是分别弹出两张互相打架的卡片。合并时恢复 / 负荷保护优先，不会把漏掉的训练量机械堆回未来几天。如果未来计划里有 **主课**，综合建议会默认先保护主课，优先调整周围的轻松课、恢复课或辅助课；如果确实建议动主课，必须把原因说清楚。

点 **3 天内不再提醒** 后，主动建议状态条会安静 3 天；这段时间里，即使你每天记录晨间状态或训练备注，也不会因为信号重新计算而反复弹。3 天后，系统会根据当时的训练、计划、晨间状态和备注重新判断。如果近期连续跳过同类主动建议，系统也会降低这类建议的自动出现频率；手动一键综合调整仍然可用。

如果系统判断当前计划偏差或恢复信号需要主动处理，AI Coach 顶部会出现综合调整提示；生成后的建议仍然需要你确认才会写入 Calendar。

如果最近一场目标赛进入 14 天内，并且赛事信息足够明确，AI Coach 上方还会出现 **赛前简报 / 装备检查**。A / B / C 级目标赛都会触发；如果同一天有多场比赛，才按 A > B > C 排优先级。它会自动整理比赛日重点、配速 / 强度策略、补给喝水、装备检查、出发前流程和本周不要做的事；室外赛事会结合比赛地点和天气，室内 Hyrox 不需要天气。点 **查看简报** 可以打开完整内容，也可以在最近建议里继续追问教练。第一版只是 briefing / checklist，不会自动改日历；如果你点 **3 天内不再提醒**，这条赛前提醒也会安静 3 天。

日历建议会默认保留「教练判断」「日历影响」「准备写入」三块重点信息，并用「日历影响」逐日说明哪些日期会新增、替换、改成休息，或只修改某一条已有计划。风险等级、整理来源和确认说明会收在右上角的 `!` 说明按钮里。只有「日历影响」里列出的日期会被改；没列出的日期会保持原计划。确认前不会改动任何数据。你可以在弹窗里逐条调整日期、类型、距离、爬升、时段等字段，也可以取消勾选或删除不想加入日历的条目。主动自动生成的建议不会把今天或过去日期放进可保存项；普通聊天 / 周报里你主动导入的计划仍可按你选择的日期处理。点**加入日历**后，选中的安排才会写入 Calendar；点**先跳过**则不会保存这次建议。点右上角关闭只会暂时关掉窗口，不会改变建议状态。

- **明确休息日会写入日历状态**：没安排的日子仍然留空；但如果教练明确说某一天休息 / 不跑 / 不安排训练，日历建议会生成「计划休息」条目。加入后这一天会显示计划休息，并清除同日旧的计划训练；已完成训练不会受影响。普通空白日只是没有活动记录，不会自动变成「计划休息」。
- **修改已有计划会尽量只改那一条**：如果 Coach 明确是在调整某条已经存在的未来计划，日历建议会把它标成「修改」。加入时只替换那条计划，不会把同一天其它上午 / 晚上计划一起清掉。只有新增计划或明确休息日，才会按日期替换同日旧计划。如果修改前后没有实质差异，它不会作为一条新的修改建议显示。
- **同日多条计划会一一对应**：判断完成情况时，每条实际训练最多匹配一条计划；同一天有两条 Road Run、但只完成一条时，不会把这一条训练同时算成两条计划都完成。系统会优先按训练类型、细分类型、目标距离 / 时长 / 爬升和上午 / 晚上匹配。
- **主课会被特别提醒**：如果建议会修改或覆盖你标成主课的计划，日历建议里会出现提醒。系统不会禁止你保存这条建议，但会要求你先确认教练给出的理由是否足够充分。
- **每种运动只填相关字段**（时段所有类型都有）：路跑 = 跑步类型 + 距离；越野跑 / 徒步 = 距离 + 爬升；爬楼 = 爬升；骑行 = 距离 + 速度；游泳 = 时长；力量 = 训练部位；HIIT = 只有时段。导入弹窗里这些都能逐条改。
- **同一条回复会记住上次整理结果**：需要重跑时点窗口上方的「重新整理这条回复」。加入后按钮会显示「已加入日历」；跳过后显示「已跳过建议」。如果保存过程中失败，建议会标记为失败，方便之后重新查看或重新整理。这些状态会保存到账号下，换设备后也能恢复。
- **旧建议不会覆盖另一台设备上的新安排**：日历建议进入审核时，会记住当时涉及日期的计划和「计划休息」状态。点「加入日历」后，App 会先刷新并核对一次；如果目标计划被修改或删除、同日计划有新增 / 减少，或计划休息状态发生变化，旧建议会立即停用，Calendar 和计划休息都不会写入。窗口会列出发生变化的日期或计划，并提供「按最新日历重新审核」入口；重新整理出的建议仍会重新展示影响范围，当前确认流程下需要再次点「加入日历」。没有这份审核记录的历史建议，也会先按最新日历建立审核依据并重新展示，不能直接执行。
- **极短的同时修改窗口仍然存在**：上面的保护发生在最终刷新与真正保存之间。如果另一台设备恰好在这两步之间的极短时间内修改同一日期，目前还不能做到数据库层面的原子拦截；因此跨设备同时编辑同一天计划时，仍建议等一端保存完成后再在另一端确认。
- **当前焦点可查看**：在 AI Coach 设置里打开 **当前焦点**，可以看到教练当前的长期焦点、短期焦点、系统正在观察哪些计划偏差 / 恢复风险 / 赛前信号，以及是否有可以生成或打开的建议卡。这里是只读汇总，不会直接改数据；要改变焦点，需要去 Races 改目标赛，或去 Calendar 改未来计划 / 完成状态。点 **查看简报** 或 **一键综合调整** 会在当前焦点上方打开对应详情；关掉详情后仍回到当前焦点。
- **最近建议可以回看和继续追问**：在 AI Coach 设置里打开 **最近教练建议**，可以查看最近 10 条日历 / 赛前简报建议，包括待确认、保存中、已保存、已跳过或失败状态、来源、涉及日期 / 数量、质量信号，以及失败原因。数据和建议记录都保存成功后，状态才会显示为“已保存”；中途失败会保留审核内容和失败状态，可安全重试。点开单条可以看内容并继续问教练；长按某条记录只会删除这条建议记录，不会回滚已经写入 Calendar 的计划。夜间 Memory 维护属于内部生命周期，不进入这个列表。
- **日期过去后会补一份执行观察**：只针对已经保存到 Calendar 的建议；等所有涉及日期都过去并刷新到最新训练数据后，最近建议会显示完成、部分完成、未完成、计划被修改 / 删除和高 RPE 数量。同日多条计划仍按一对一规则匹配。这些结果会作为后续 AI Coach 和周复盘的观察事实，但不会写成“这条建议导致了某个结果”。重复刷新只更新同一份观察，不会新增重复记录，也不会覆盖原来的保存审计。
- **计划时长只到分钟 / 小时**，不显示秒。

导入的项目在日历上以**虚线框**显示，**不**计入统计或 PR 栏。

AI Coach 的计划导入分两种情况：新增计划 / 明确休息日会按日期替换当天已有的**计划训练**，避免同一天叠出两份互相冲突的计划；修改已有计划时只替换被标记的那一条。已经完成的训练不会被删除或覆盖。

> 不走 AI 导入也行：日历任意一天点 **Plan a workout** 手动排，字段同样按运动类型显示。保存过训练位置后，计划表单里可以选择这次训练准备去哪里跑；一周里最想保住的关键课可以勾选 **主课**。补到历史日期，或中午后补今天上午的计划时，会先提示这是在补录计划；早上安排今晚的计划不会提示。

### 计划完成怎么判定

过去（和今天已完成）的计划，会自动跟当天**同一类型**的实际训练对照：

- 比对计划目标和实际做到的量（距离类比距离、爬楼比爬升、游泳 / 力量比时长）：实际达到目标 **80% 以上**算**完成**（跑多了当然也算）；做了同类型但没到 80% 算**部分完成**；当天没有同类型训练算**漏掉**。
- HIIT 这类没有具体目标量的，当天有同类型训练即算完成。
- 自动判定不满意时，日详情里每条计划都能手动点**已完成 / 撤销**，手动结果优先。

日历格子里，过去（和今天已完成）的计划会收进「+N 计划」的小标，点开当天详情能看到每条计划和它的完成情况；未来的计划仍直接显示在格子里。同一天有多条未来计划时，会按上午、未指定、晚上的顺序排列。

> 不想看到这个按钮？在 AI Coach 设置里把「日历建议按钮」切到 **Hide** 就行。关掉后所有回复下面都不再显示这个按钮 —— 想用时随时回来打开。

## 教练偏好 / Memory / Prompt 预览

电脑端和手机端都从 AI Coach 输入框旁边的 **⚙** 打开。常用项会直接显示：当前焦点、教练偏好、训练偏好、Memory、最近教练建议；Prompt 预览、日历建议按钮和清空对话收进「高级」。

对话框上方会显示精简状态栏和一条可点击的 **当前焦点** 摘要。摘要会显示最近目标赛 / 下一条计划和需要留意的信号；点开后可查看长期焦点、短期焦点、观察项和可确认的下一步。

### 教练偏好

三个维度各三档：

- **风格**：温和鼓励 / 平衡理性 / 严格数据驱动
- **输出长度**：极简 / 标准 / 详细
- **风险提醒强度**：轻提醒 / 标准 / 严格监督

随时可改，下一条消息开始生效。

### Preview Prompt

预览的是发给 AI 的**结构**，不是完整内容：核心教练指令是产品专有的，**隐藏**不显示；你的各类数据（资料 / 目标赛事 / 最近训练 / 天气等）只列出**有哪些类别**（占位符形式），不显示具体数值。这样你能看清每次都发了哪些信息，又不暴露写死的提示词和你的原始数据。预览有 EN / 中 切换 —— **英文版才是真正发给 AI 的**（英文指令模型执行更稳定），中文仅供你阅读理解。

> 实际发出去的是带真实数据的完整提示词，预览只是"打了码"的骨架。

如果 Coach 说"看不到你最新的训练"，那基本是它在**装睡** —— 最近训练这个数据块**每条消息都会附上**（你能在预览的结构里看到这一项）。直接反问让它再看一遍即可。
