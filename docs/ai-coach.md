# AI 教练

每日打卡式的对话教练，基于你的真实训练数据、目标赛事和长期记忆给个性化建议。

## 初次配置

不需要自己申请或填写第三方服务密钥。

1. 打开 **AI Coach** tab 直接发消息。AI Coach 会自动使用 DeepSeek，不需要选择模型。
2. 每次 AI 对话、AI 导入日历、自动更新 Memory 都会记录一次内部成本；AI 按 DeepSeek 实际模型成本以人民币结算，四舍五入到分，最低 ¥0.01。
3. 当前是个人使用模式，不显示钱包入口、充值入口和管理员充值入口。
4. 对话历史会自动保存，刷新和换设备都不会丢。

每条 AI 回复下方会显示本次模型、token 总量和内部成本。DeepSeek 还会拆出缓存命中 / 普通输入 / 输出 token；如果大部分输入命中缓存，看到一万多 token 但成本仍然很低是正常的，因为缓存命中的输入单价很低。

## 教练每次能看到什么

每次你发消息，教练都会拿到这些信息：

- **个人资料** —— 年龄、性别、城市、职业、训练年限、伤病史、可用器材，以及（如果填了 Resting HR + Max HR）个人心率区间。
- **教练配置** —— 三个轴（风格、输出长度、风险提醒强度），可在 Coach Config 面板调。
- **长期记忆（Memory）** —— 你自己（或 AI 自动）维护的关于你的持久事实。
- **当前时间**和**全部目标赛事**。
- **最近一场目标赛事的比赛日天气**（仅当你给那场赛事填了地点；室内的 Hyrox 不算）—— 临近比赛（约两周内）是真实预报，还早则是该地点该日期的往年同期典型天气。教练据此给比赛日的配速 / 补水 / 装备建议、提前提示高温高湿。只带**最近这一场**，后面几个月的比赛不会一并念出来。
- **精选历史赛事**（每个项目只发最近一场 + 最好成绩那场）。
- **最近 8 周的周训练量趋势** —— 每周一行：跑步类（Road Run / Trail Run / Hiking / Floor Climbing）的总距离、总爬升、训练次数。让教练能看出"这周比上周猛涨"这类周期化 / 伤病风险信号，而不只是孤立的最近 10 条。只统计跑步类训练；没有任何跑步数据的周（以及开头连续的空周）不显示。
- **训练负荷与骤增风险** —— 用最近 7 天和最近 4 周的「时长 × RPE」算出你的负荷强度比值（ACWR）。教练据此判断你是不是加得太猛、要不要收一收。攒够约 4 周数据后才准（不够时显示"基线建立中"）。
- **最近几天的晨间状态自评** —— 你在日历里打的睡眠 / 腿感 / 精力，让教练判断今天该顶还是该收。
- **计划依从情况** —— 最近两周你排的计划训练，完成了几次、部分完成几次、漏了几次、主动跳过几次（按同类型实际训练对照计划目标判定）。漏了的，教练会先问你怎么回事，而不是闷头继续加量。
- **最近 10 条已完成训练**（计划训练不包括在内）—— 含你填的 **RPE（自觉用力）**和**备注**，行尾还会附上当时的天气（如果有）。
- **最近 21 天的当日标记** —— 你在日历上勾的 按摩 / 拉伸 / 睡眠差 / 生病 / 出差。教练排计划、评估负荷时会参考（生病、出差尤其关键）。没勾过就不出现。
- **当前天气**（温度、体感、湿度、天气状况、风速、AQI）—— 实时拉取。
- **未来 7 天的天气预报**（每天一行：最高/最低、体感、湿度、天气状况、风速、AQI）—— 无论你那天有没有训练计划，预报都会发给教练，让它在排新计划时主动按天气挑日子。
- **今天和未来约 21 天的计划训练** —— 你排在日历上的计划（不限于一周内）都会发给教练，**所以你可以直接让它评估计划的可行性**（"这周这样排合不合理""这个 long run 安排得对吗"）。今天的计划会明确标成「只是已安排，不代表已经完成」，避免教练把今晚还没发生的训练当作你已经做完。7 天内的计划会附上当天预报。
- **本次教练重点** —— 系统会按你当下的情况，临时给教练加一两条提醒：临近目标赛时提示当前处于堆量 / 巅峰 / 减量哪个阶段；比赛日高温高湿时提醒给热适应建议；负荷骤增时提醒别再硬加；最近漏了计划时提醒先问原因。没触发就不加。
- **完整对话历史**。

> 教练不会主动获取你浏览器之外的任何数据。你给它什么，它就用什么。

### 关于训练数据的新鲜度

每次发消息**之前**都会重新从数据库拉一次最新训练记录，所以无论你是刚在同一个窗口里加的训练，还是在另一台设备上录的训练，下一条 Coach 回复都能看到。

加完一条训练立刻问 "我刚跑完帮我看看"，Coach 应该直接引用那条的距离/时长/心率 —— 如果它说"我看不到你最新的数据"或者"系统还没同步"，那是 LLM 在偷懒**幻觉**，把它当作错答处理，直接反问 "5/24 的数据应该有，请再看一下"，它就会承认。

手动添加或导入活动后，应用会弹出一个确认框。点 **发送给 AI 教练** 后，系统会自动把这次活动发到 AI Coach，并打开 AI Coach 等回复；你不需要再跳过去手动输入或发送。点取消则不会消耗 AI 对话额度。

### 你在对话里说的话，优先于日历计划

如果你直接告诉 Coach 一个当下的变更 —— 比如 "我今晚要训练"、"这次长距离我不跑了"、"周六那次挪到周日" —— 它会**以你刚说的为准**，而不是继续按日历里那条旧计划安排。日历计划只是默认值，不是约束。所以想让它换个思路，最快的方式就是直接在对话里说清楚。

## Memory（长期记忆）

一段你（或 AI）随时间维护的自由文本。这是「持久层」—— 比如：

- 「轻松日必须真的轻松，不然跟腱会犯」
- 「备战 2026 UTMB CCC，8 月底 A 级比赛」
- 「不喜欢被人在恢复跑里提速度」

两种更新方式：

- **Edit** —— 打开 Memory，点 Edit，自己写，Save。**你手动写的内容（中文或英文都行）就是真正发给 AI 的**，不会被忽略。
- **Auto-update from chat** —— 把当前 memory + 最近对话发给 AI，让它产出一份建议更新版（中英成对）。你审核后选 Accept 或 Discard。建议内容按你的语言设置显示（面板里也能中/EN 切换）。

AI 在生成建议更新时被要求**只保留持久事实**，丢掉一次性琐事，控制在大约 500 字以内。建议版会按主题整理，例如伤病 / 健康、目标 / 比赛、训练偏好、教练风格和长期模式，避免所有记忆挤成一段后越用越乱。自动建议会同时生成英文和中文；正式发给 Coach 的主记忆是英文版，中文版用于你在面板里审核和阅读。

更新过程在**后台进行**：点了更新后即使切到别的页、关掉记忆面板也没关系，它会继续跑（更新中按钮有转圈动画）。跑完 App **顶部会弹出一条通知**，点一下就回到记忆面板，审核要保留 / 删除的条目。

## 长对话软提示

聊累计到 **40 条**消息时，对话上方会出一条琥珀色软提示，建议你把要点固化进 Memory + 清空对话；提示里直接有**"清空聊天"**按钮，点它清空前会再提醒你先更新一下记忆。原因是历史对话越长，旧消息越跟系统提示词争抢 AI 的注意力；定期固化进 Memory 能保持回复质量。

点提示上的按钮直接跳到 Memory 区域。自动更新 Memory 也属于一次 AI 调用，会记录内部成本。

## 把建议导入到日历

每条 AI 回复下方都有一个 **📅 Import to Calendar** 按钮（默认显示，可在设置里关掉）。点它会让 AI 把那条回复里的训练建议解析成结构化的计划项目，弹一个 review modal 让你逐条调整后再导入。首次提炼会记录一次内部成本。

- **休息日会空着**：只有教练明确安排了训练的那天才会生成计划项，没安排的日子留空（不会强行每天塞一个任务）；只有教练明确给了恢复性训练才会标成 Recovery。
- **每种运动只填相关字段**（时段所有类型都有）：路跑 = 跑步类型 + 距离；越野跑 / 徒步 = 距离 + 爬升；爬楼 = 爬升；骑行 = 距离 + 速度；游泳 = 时长；力量 = 训练部位；HIIT = 只有时段。导入弹窗里这些都能逐条改。
- **同一条回复会记住上次提炼结果**：如果你已经点过导入并成功提炼，再点同一条回复的 **Import to Calendar** 会直接打开上次的审核窗口，不会重复消耗一次提炼；需要重跑时点窗口上方的「重新提炼这条回复」。
- **计划时长只到分钟 / 小时**，不显示秒。

导入的项目在日历上以**虚线框**显示，**不**计入统计或 PR 栏。

> 不走 AI 导入也行：日历任意一天点 **Plan a workout** 手动排，字段同样按运动类型显示。补到历史日期，或下午补今天上午的计划时，会先提示这是在补录计划；早上安排今晚的计划不会提示。

### 计划完成怎么判定

过去（和今天已完成）的计划，会自动跟当天**同一类型**的实际训练对照：

- 比对计划目标和实际做到的量（距离类比距离、爬楼比爬升、游泳 / 力量比时长）：实际达到目标 **80% 以上**算**完成**（跑多了当然也算）；做了同类型但没到 80% 算**部分完成**；当天没有同类型训练算**漏掉**。
- HIIT 这类没有具体目标量的，当天有同类型训练即算完成。
- 自动判定不满意时，日详情里每条计划都能手动点**已完成 / 跳过 / 撤销**，手动结果优先。

日历格子里，过去（和今天已完成）的计划会收进「+N 计划」的小标，点开当天详情能看到每条计划和它的完成情况；未来的计划仍直接显示在格子里。

> 不想看到这个按钮？在 AI Coach 设置里把「Calendar Import Button」切到 **Hide** 就行。关掉后所有回复下面都不再显示这个按钮 —— 想用时随时回来打开。

## Coach Config / Memory / Prompt 预览

电脑端这三块在 AI Coach tab 的顶部按钮区展开；手机端则点输入框旁边的 **⚙** 进入设置子页打开。

对话框上方会显示一排状态 chip：当前模型、教练模式、Memory 是否已设置、日历导入按钮是否显示。调整设置后这里会同步更新，方便发消息前快速确认当前教练状态。

### Coach Config（教练行为）

三个维度各三档：

- **风格**：温和鼓励 / 平衡理性 / 严格数据驱动
- **输出长度**：极简 / 标准 / 详细
- **风险提醒强度**：轻提醒 / 标准 / 严格监督

随时可改，下一条消息开始生效。

### Preview Prompt

预览的是发给 AI 的**结构**，不是完整内容：核心教练指令是产品专有的，**隐藏**不显示；你的各类数据（资料 / 目标赛事 / 最近训练 / 天气等）只列出**有哪些类别**（占位符形式），不显示具体数值。这样你能看清每次都发了哪些信息，又不暴露写死的提示词和你的原始数据。预览有 EN / 中 切换 —— **英文版才是真正发给 AI 的**（英文指令模型执行更稳定），中文仅供你阅读理解。

> 实际发出去的是带真实数据的完整提示词，预览只是"打了码"的骨架。

如果 Coach 说"看不到你最新的训练"，那基本是它在**装睡** —— 最近训练这个数据块**每条消息都会附上**（你能在预览的结构里看到这一项）。直接反问让它再看一遍即可。

## 提示

- AI Coach 使用内置 DeepSeek，不需要你自己申请或保存第三方服务密钥；回复下方会显示本次调用的 token 和内部成本。
- **长按教练消息**可以选中、复制文字。
- 如果对话框显示共享余额不足，需要先在后台补充余额；如果显示 "No response."，通常是服务端或模型临时异常，稍后重试。
- 模型或网络临时错误显示为对话里的临时气泡，**不**写入历史。
- **安卓 APP** 在等回复时切到后台再回来不会再被打断 —— 对话请求走的是系统原生网络层，不依赖 APP 的前台状态。
- **网页版**（特别是 iOS Safari）切走仍可能让请求中断；回来后点用户消息下方的 **↻ 重新发送** 即可重试。
