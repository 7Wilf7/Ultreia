# AI 教练

每日打卡式的对话教练，基于你的真实训练数据、目标赛事和长期记忆给个性化建议。

## 初次配置

不需要自己申请或填写第三方服务密钥。

1. 打开 **AI Coach** tab 直接发消息。默认会优先使用本机 Codex runner；如果 Codex runner 不可用、超时或返回失败，会自动回退到 DeepSeek。
   模型状态前的小圆点表示 Codex runner 状态：绿色 = 下一次预计调用 Codex，橙色 = 正在检查或短暂失去心跳，红色 = 离线、认证异常或上次调用失败。即使变红，AI Coach 仍会尝试自动改用 DeepSeek。这个状态会在进入页面时检查一次、页面保持打开时约每 5 秒刷新一次、从后台回到前台时再刷新一次；电脑关机或断网后通常会在约 20-30 秒内变成离线。
2. AI Coach 对话、AI 周复盘、计划提炼 / 补救、Memory 更新和每日推送都会走这条 AI 路线。
3. 你在界面里看到的 AI 输出默认使用中文，包括日常对话、每日教练推送、AI 周复盘、日历建议和赛前简报。长期记忆建议仍会同时生成英文和中文：英文用于后续喂给教练，中文用于你审核。
4. 当前是个人使用模式，AI 调用不检查钱包余额，也不会扣钱包；界面不显示钱包入口、充值入口和管理员充值入口。
5. 对话历史会自动保存，刷新和换设备都不会丢。
6. 手机端输入框右侧有图片按钮和发送按钮；输入框会和这两个按钮贴齐，输入多行时才向上展开。
7. 输入框右侧的图片按钮可以给教练附图，例如训练计划截图、装备照片或体感相关图片。图片会在发送前压缩，只用于本次 Codex 分析；聊天记录里只保存“本次附图”的文字标记，不保存原图。为了交给本机 Codex，图片会短暂进入 AI 任务队列；任务被 runner 领取、完成、失败或超时后，队列里会清理成只含图片名和类型的元数据。DeepSeek 看不了图片，所以带图请求会强制走 Codex；如果 Codex runner 当前不可用，系统会提示重试或改成文字描述，不会把图片请求静默回退到 DeepSeek。
8. AI Coach 顶部的定位图标用于设置天气默认地点。顶部只显示城市缩写，比如广州显示 GZ；点开后可看地图预览、用当前位置或搜索地点名，不需要去个人资料里找；天气和教练的天气上下文都会按这里的默认地点刷新。

## 教练每次能看到什么

每次你发消息，教练都会拿到这些信息：

- **个人资料** —— 年龄、性别、职业、训练年限、伤病史、可用器材，以及（如果填了 Resting HR + Max HR）个人心率区间。
- **教练偏好** —— 三个轴（风格、输出长度、风险提醒强度），可在教练设置里用下拉列表调整。
- **长期记忆（Memory facts）** —— 你审核过的持久事实卡片；Current facts 会进入 AI Coach 和 AI 周复盘上下文，Archived facts 不会进入。
- **当前时间**和**全部目标赛事**。
- **最近一场目标赛事的比赛日天气**（仅当你给那场赛事填了地点；室内的 Hyrox 不算）—— 临近比赛（约两周内）是真实预报，还早则是该地点该日期的往年同期典型天气。教练据此给比赛日的配速 / 补水 / 装备建议、提前提示高温高湿。只带**最近这一场**，后面几个月的比赛不会一并念出来。
- **精选历史赛事**（每个项目只发最近一场 + 最好成绩那场）。
- **最近 8 周的周训练量趋势** —— 每周一行：跑步类（Road Run / Trail Run / Hiking / Floor Climbing）的总距离、总爬升、训练次数。让教练能看出"这周比上周猛涨"这类周期化 / 伤病风险信号，而不只是孤立的最近 10 条。只统计跑步类训练；没有任何跑步数据的周（以及开头连续的空周）不显示。
- **训练负荷与骤增风险** —— 用最近 7 天和最近 4 周的「时长 × RPE」算出你的负荷强度比值（ACWR）。教练据此判断你是不是加得太猛、要不要收一收。攒够约 4 周数据后才准（不够时显示"基线建立中"）。
- **最近几天的晨间状态自评** —— 你在日历里打的睡眠 / 腿感 / 精力，让教练判断今天该顶还是该收。
- **计划依从情况** —— 最近两周你排的计划训练，完成了几次、部分完成几次、漏了几次（按同类型实际训练对照计划目标判定）。漏了的，教练会先问你怎么回事，而不是闷头继续加量。
- **最近 10 条已完成训练**（计划训练不包括在内）—— 含你填的 **RPE（自觉用力）**和**备注**，行尾还会附上当时的天气（如果有）。
- **最近 21 天的当日标记** —— 你在日历上勾的 计划休息 / 按摩 / 拉伸 / 生病。教练排计划、评估负荷时会参考。没勾过就不出现。
- **当前天气**（温度、体感、湿度、天气状况、风速、AQI）—— 实时拉取。
- **未来 7 天的天气预报**（每天一行：最高/最低、体感、湿度、天气状况、风速、AQI）—— 无论你那天有没有训练计划，预报都会发给教练，让它在排新计划时主动按天气挑日子。
- **今天和未来约 21 天的计划训练** —— 你排在日历上的计划（不限于一周内）都会发给教练，**所以你可以直接让它评估计划的可行性**（"这周这样排合不合理""这个 long run 安排得对吗"）。今天的计划会明确标成「只是已安排，不代表已经完成」，避免教练把今晚还没发生的训练当作你已经做完。7 天内的计划会附上当天预报。
- **最近几条教练建议反馈** —— 比如哪些日历建议你确认了、跳过了、保存失败了，以及 Memory 是否已保存。教练会把这些当成反馈，避免反复提出你已经跳过的方向。
- **本次教练重点** —— 系统会按你当下的情况，临时给教练加一两条提醒：临近目标赛时提示当前处于堆量 / 巅峰 / 减量哪个阶段；比赛日高温高湿时提醒给热适应建议；负荷骤增时提醒别再硬加；最近漏了计划时提醒先问原因。没触发就不加。
- **完整对话历史**。

> 教练不会主动获取你浏览器之外的任何数据。你给它什么，它就用什么。

### 关于训练数据的新鲜度

每次发消息**之前**都会重新从数据库拉一次最新训练记录，所以无论你是刚在同一个窗口里加的训练，还是在另一台设备上录的训练，下一条 Coach 回复都能看到。

加完一条训练立刻问 "我刚跑完帮我看看"，Coach 应该直接引用那条的距离/时长/心率 —— 如果它说"我看不到你最新的数据"或者"系统还没同步"，那是 LLM 在偷懒**幻觉**，把它当作错答处理，直接反问 "5/24 的数据应该有，请再看一下"，它就会承认。

手动添加或导入活动后，应用会弹出一个确认框。点 **发送给 AI 教练** 后，系统会自动把这次活动发到 AI Coach，并打开 AI Coach 等回复；你不需要再跳过去手动输入或发送。点取消则不会调用 AI。导入活动时，会先等活动保存成功，再弹出发送给教练的确认框，避免教练看不到刚导入的数据。如果你在导入弹窗里写了主观感受，聊天里会显示这句自然备注；后台仍会把新增活动、当前计划、训练负荷、计划依从和近期教练建议一起发给 Coach。教练会同时点评这次训练，并对下一次相关计划给出一个明确口径：保留 / 降量 / 取消 / 暂不判断。点评完成后，应用会结合你的原始感受和教练点评，压缩成一条「自评」备注写回训练，之后复盘和后续 Coach 对话都能看到。

为了避免刚点评完一条新训练、主动调整又立刻给出另一套口径，导入后主动发送给教练点评时，应用会把当下的计划偏差 / 恢复负荷守门 / 综合调整签名记为“已处理”，并让自动生成的主动调整提醒暂时安静。同一批状态不会马上再次弹出另一张建议卡；如果你仍想主动调整，可以去 AI Coach 设置的 **动作** 分组里手动触发。

### 你在对话里说的话，优先于日历计划

如果你直接告诉 Coach 一个当下的变更 —— 比如 "我今晚要训练"、"这次长距离我不跑了"、"周六那次挪到周日" —— 它会**以你刚说的为准**，而不是继续按日历里那条旧计划安排。日历计划只是默认值，不是约束。所以想让它换个思路，最快的方式就是直接在对话里说清楚。

## Memory facts（长期记忆）

长期记忆现在以事实卡片保存。这是「持久层」—— 比如：

- 「轻松日必须真的轻松，不然跟腱会犯」
- 「备战 2026 UTMB CCC，8 月底 A 级比赛」
- 「不喜欢被人在恢复跑里提速度」

在 AI Coach 的设置里打开 **Memory**，会进入一个完整的长期记忆页面，可以看到 **Current** 和 **Archived** 两类事实，并用 **全部 / 伤病健康 / 目标比赛 / 训练偏好 / 教练风格 / 长期模式** 这些 chip 快速筛选。

- **Current** —— 当前启用的事实，会进入 AI Coach 和 AI 周复盘上下文。
- **Archived** —— 已归档事实，仍可查看、恢复或删除，但不会进入 Prompt。删除前会再次确认，删除后无法恢复。

更新方式：

- **从当前对话更新** —— 按钮在长期记忆标题右侧。它会把当前 Memory facts + 最近对话发给 AI，让它产出一份建议更新。结果会以确认卡片形式出现：你先勾选要保留 / 删除的条目，再点 Accept 才会启用为 Current facts；点 Discard 不会改动任何内容。
- **Nightly Memory review（夜间记忆审核）** —— 开启后，如果当天有新的 AI Coach 对话，系统会在夜间最多整理一次记忆建议。它只会生成一张待确认的记忆建议和收件箱提醒，不会自动启用事实；第二天打开 App 后，顶部会提示你确认，只有点 Accept 才会保存。这个过程会调用一次 AI，但不需要余额或充值。

AI 在生成建议更新时被要求输出一份**完整、去重后的当前记忆快照**，而不是只追加新条目。你点 Accept 后，应用会尽量匹配已有事实：相近内容会更新原卡片；快照里不再出现的旧事实会自动移到 Archived；真正新内容才会新增为 Current facts。

建议只保留持久事实，丢掉一次性琐事，控制在大约 500 字以内。分类边界会收紧为：伤病 / 健康只放伤病、恢复、疲劳准备度和风险指标；目标 / 比赛只放当前目标赛事；训练偏好放计划结构、器械、地形和训练方式；长期模式放反复出现的行为模式；教练风格只放你希望教练如何沟通和判断。建议会同时生成英文和中文；英文用于模型上下文，中文用于你在面板里审核和阅读。

更新过程在**后台进行**：点了更新后即使切到别的页、关掉记忆面板也没关系，它会继续跑（更新中按钮有转圈动画）。跑完 App **顶部会弹出一条通知**，点一下就回到记忆面板，审核要保留 / 删除的条目。

Memory 的自动更新和日历建议一样，都是「确认前不改数据」：AI 只负责整理建议，真正保存发生在你点 Accept 之后。保存成功后，这些事实会写入账号并能在不同设备同步；如果保存失败，App 会保留确认卡片并提示错误，不会把这次更新标记为已保存。旧的整段文本 Memory 已退出 Prompt，AI Coach 和 AI 周复盘只读取 Current Memory facts。

## 收件箱与主动复盘

AI Coach 页顶部的信封是应用内收件箱；旁边的齿轮会打开教练设置。常用入口会先显示 **教练偏好 / Memory / 编辑资料**；**一键综合调整** 和已有记录时的 **最近教练建议** 放在 **动作** 分组；Prompt 预览、日历建议按钮和清空对话收在 **高级** 里。每日教练打卡、AI 周复盘完成提醒、Memory 审核提醒这类主动消息都会保存在收件箱里；即使系统通知被划掉，也能回到收件箱查看。

每日教练推送在 Settings 里设置时间，只在安卓 App 上真正弹系统通知，推送正文默认中文。开启后，安卓通知栏会出现一个低优先级、静音的 Ultreia 常驻通知，用来提高系统保留推送连接的概率；通知内容会显示本月的 Sessions / Time 和 Distance / Ascent，并拆成两行对齐显示。这里的 Distance 和 Training 顶部本月 Distance 口径一致，只统计 Road Run / Trail Run / Hiking / Floor Climbing 的距离；HIIT / Strength 即使导入文件里带了距离，也不会算进通知栏 Distance。如果手机系统的电池限制很严格，仍然建议把 Ultreia 加进电池/后台运行白名单。

AI 周复盘从 Settings 进入，是一个完整页面，不是弹窗。你可以选择本周或上周，手动点击分析数据；周报会读取对应时间范围内的完成训练、计划、晨间状态、目标赛事和 Current Memory facts，默认生成中文报告。报告会逐条点评每一次训练，指出本周风险和恢复重点，并给出接下来 7 天的训练安排。

点 **分析数据** 后可以离开周报页或切到别的 App，生成完成后安卓会弹系统通知；回到 Settings → AI 周复盘 → 详情 就能看到新报告。分析中，Settings 列表里的 AI 周复盘会显示“正在分析”和转圈动画；如果正在从周报提炼计划，会显示“正在提炼”。本周和上周两个标签会各自保留最新一份账号内周报，换设备也能查看。

Settings 里点击 **AI 周复盘** 后会展开 **详情** 和 **设置**。进入设置后，可以开启自动生成，用滚轮选择星期几和具体时间；服务器会按手机保存的本地时区，在所选时间每周自动生成一次本周周报，即使 Ultreia 没有打开也会运行。完成后会写入账号里的周报页面，并发送系统通知 / 收件箱提醒。

“周日导入后补漏”也可以在这里开关。如果服务器已经生成过本周周报、之后你又在周日导入了新活动，App 会询问是否立即重新分析，避免周报漏掉刚导入的训练；如果预定生成时间已过但周报仍未生成，也会提供手动分析兜底。预定时间还没到时不会重复打扰。

周报标题右侧有小号的 **分析** 和 **导入计划** 按钮。点击 **导入计划** 会把报告里的具体安排整理成日历建议，你仍然可以逐条修改 / 取消 / 确认后再写入 Calendar；确认前不会自动改日历。分析或提炼正在运行时，按钮会变成 **停止**，点一下会停止当前等待并忽略后续返回结果。

如果周报里有需要你补充的问题，可以在报告底部输入回答并发送到 AI Coach。App 会带上这份周报和你的补充，跳到 AI Coach 继续对话；这个动作不会自动改日历或 Memory。

## 长对话软提示

聊累计到 **40 条**消息时，对话上方会出一条琥珀色软提示，建议你把要点固化进 Memory facts + 清空对话；提示里直接有**"清空聊天"**按钮，点它清空前会再提醒你先更新一下记忆。原因是历史对话越长，旧消息越跟系统提示词争抢 AI 的注意力；定期固化进 Memory facts 能保持回复质量。

点提示上的按钮直接跳到 Memory 区域。自动更新 Memory facts 也会调用一次 AI，但不需要余额或充值。

## 把建议导入到日历

每条 AI 回复下方都有一个 **看看日历建议** 按钮（默认显示，可在设置里关掉）。点它会让 AI 把那条回复里的训练建议整理成一份日历草案，目前支持的是「加入日历安排」。首次整理会调用一次 AI；同一条回复已经整理过时，按钮会变成 **查看上次整理**，再次点击会直接打开上次结果。

如果最近两周出现漏练 / 部分完成，或近期负荷、RPE、晨间状态、训练备注、生病标记显示恢复风险，AI Coach 对话上方会出现一条主动建议状态条。它会自动生成一份日历建议，不需要你先点“生成”。生成完成后点 **查看建议** 就能打开；确认前仍然不会改动日历，你可以逐条修改、取消或跳过。如果你刚把新增活动发给教练点评过，同一批计划偏差或恢复风险会被视为已处理，避免短时间内重复生成相反建议。

如果计划偏差和恢复风险同时出现，AI Coach 会合并成一份综合建议，而不是分别弹出两张互相打架的卡片。合并时恢复 / 负荷保护优先，不会把漏掉的训练量机械堆回未来几天。

点 **3 天内不再提醒** 后，主动建议状态条会安静 3 天；这段时间里，即使你每天记录晨间状态或训练备注，也不会因为信号重新计算而反复弹。3 天后，系统会根据当时的训练、计划、晨间状态和备注重新判断。

如果你不想等自动提醒，也可以在 AI Coach 设置的 **动作** 分组里点 **一键综合调整**，主动让教练根据当前计划偏差和恢复信号生成一份日历建议。手动点击后会先弹出二次确认，避免误触；生成中可以点 **停止**，停止后不会继续打开建议卡片。这个入口不受 3 天静默影响；生成后的建议仍然需要你确认才会写入 Calendar。

如果最近一场目标赛进入 14 天内，并且赛事信息足够明确，AI Coach 上方还会出现 **赛前简报 / 装备检查**。A / B / C 级目标赛都会触发；如果同一天有多场比赛，才按 A > B > C 排优先级。它会自动整理比赛日重点、配速 / 强度策略、补给喝水、装备检查、出发前流程和本周不要做的事；室外赛事会结合比赛地点和天气，室内 Hyrox 不需要天气。点 **查看简报** 可以打开完整内容，也可以在最近建议里继续追问教练。第一版只是 briefing / checklist，不会自动改日历；如果你点 **3 天内不再提醒**，这条赛前提醒也会安静 3 天。

日历建议会默认保留「教练判断」「日历影响」「准备写入」三块重点信息，并用「日历影响」逐日说明哪些日期会新增、替换、改成休息，或只修改某一条已有计划。风险等级、整理来源和确认说明会收在右上角的 `!` 说明按钮里。只有「日历影响」里列出的日期会被改；没列出的日期会保持原计划。确认前不会改动任何数据。你可以在弹窗里逐条调整日期、类型、距离、爬升、时段等字段，也可以取消勾选或删除不想加入日历的条目。点**加入日历**后，选中的安排才会写入 Calendar；点**先跳过**则不会保存这次建议。点右上角关闭只会暂时关掉窗口，不会改变建议状态。

- **明确休息日会写入日历状态**：没安排的日子仍然留空；但如果教练明确说某一天休息 / 不跑 / 不安排训练，日历建议会生成「计划休息」条目。加入后这一天会显示计划休息，并清除同日旧的计划训练；已完成训练不会受影响。普通空白日只是没有活动记录，不会自动变成「计划休息」。
- **修改已有计划会尽量只改那一条**：如果 Coach 明确是在调整某条已经存在的未来计划，日历建议会把它标成「修改」。加入时只替换那条计划，不会把同一天其它上午 / 下午计划一起清掉。只有新增计划或明确休息日，才会按日期替换同日旧计划。
- **每种运动只填相关字段**（时段所有类型都有）：路跑 = 跑步类型 + 距离；越野跑 / 徒步 = 距离 + 爬升；爬楼 = 爬升；骑行 = 距离 + 速度；游泳 = 时长；力量 = 训练部位；HIIT = 只有时段。导入弹窗里这些都能逐条改。
- **同一条回复会记住上次整理结果**：需要重跑时点窗口上方的「重新整理这条回复」。加入后按钮会显示「已加入日历」；跳过后显示「已跳过建议」。如果保存过程中失败，建议会标记为失败，方便之后重新查看或重新整理。这些状态会保存到账号下，换设备后也能恢复。
- **教练建议矩阵可查看**：在 AI Coach 设置里打开 **教练建议矩阵**，可以看到当前正在推进的建议卡片类型、触发条件、会整理什么、边界和状态。这个页面是只读说明，不是自动执行开关；所有会改日历或 Memory 的建议仍然需要你确认。
- **最近建议可以回看和继续追问**：在 AI Coach 设置里打开 **最近教练建议**，可以查看最近 10 条计划导入 / Memory 更新 / 赛前简报建议，包括待确认、已确认、已保存、已跳过或失败状态、来源、涉及日期 / 数量，以及失败原因。新建议会直接出现在列表里，不需要刷新页面。点开单条可以看可读的建议内容和保存结果；如果是修改已有计划，会按日期显示「原计划 / 新计划」，隐藏内部计划编号，方便确认教练到底记了什么、保存了什么。展开记录后可以带着这条建议继续问教练，让它复盘这次建议是否合理、下一步该继续还是调整；这只会发送一条聊天消息，不会自动保存新内容。长按某条记录可以删除这条建议记录和对应消息下方的已整理状态；这不会回滚已经写入 Calendar 的计划，也不会删除已经保存成功的 Memory facts。
- **计划时长只到分钟 / 小时**，不显示秒。

导入的项目在日历上以**虚线框**显示，**不**计入统计或 PR 栏。

AI Coach 的计划导入分两种情况：新增计划 / 明确休息日会按日期替换当天已有的**计划训练**，避免同一天叠出两份互相冲突的计划；修改已有计划时只替换被标记的那一条。已经完成的训练不会被删除或覆盖。

> 不走 AI 导入也行：日历任意一天点 **Plan a workout** 手动排，字段同样按运动类型显示。补到历史日期，或下午补今天上午的计划时，会先提示这是在补录计划；早上安排今晚的计划不会提示。

### 计划完成怎么判定

过去（和今天已完成）的计划，会自动跟当天**同一类型**的实际训练对照：

- 比对计划目标和实际做到的量（距离类比距离、爬楼比爬升、游泳 / 力量比时长）：实际达到目标 **80% 以上**算**完成**（跑多了当然也算）；做了同类型但没到 80% 算**部分完成**；当天没有同类型训练算**漏掉**。
- HIIT 这类没有具体目标量的，当天有同类型训练即算完成。
- 自动判定不满意时，日详情里每条计划都能手动点**已完成 / 撤销**，手动结果优先。

日历格子里，过去（和今天已完成）的计划会收进「+N 计划」的小标，点开当天详情能看到每条计划和它的完成情况；未来的计划仍直接显示在格子里。同一天有多条未来计划时，会按上午、未指定、下午的顺序排列。

> 不想看到这个按钮？在 AI Coach 设置里把「日历建议按钮」切到 **Hide** 就行。关掉后所有回复下面都不再显示这个按钮 —— 想用时随时回来打开。

## 教练偏好 / Memory / Prompt 预览

电脑端和手机端都从 AI Coach 输入框旁边的 **⚙** 打开。常用项会优先显示：教练偏好、长期记忆、编辑资料；动作类入口放在第二组；Prompt 预览、日历建议按钮和清空对话收进「高级」。

对话框上方会显示一排状态 chip：Codex runner 状态点、当前模型、教练模式、Memory 是否已设置、日历导入按钮是否显示。调整设置后这里会同步更新，方便发消息前快速确认当前教练状态。

### 教练偏好

三个维度各三档：

- **风格**：温和鼓励 / 平衡理性 / 严格数据驱动
- **输出长度**：极简 / 标准 / 详细
- **风险提醒强度**：轻提醒 / 标准 / 严格监督

随时可改，下一条消息开始生效。

### Preview Prompt

预览的是发给 AI 的**结构**，不是完整内容：核心教练指令是产品专有的，**隐藏**不显示；你的各类数据（资料 / 目标赛事 / 最近训练 / 天气等）只列出**有哪些类别**（占位符形式），不显示具体数值。这样你能看清每次都发了哪些信息，又不暴露写死的提示词和你的原始数据。预览有 EN / 中 切换 —— **英文版才是真正发给 AI 的**（英文指令模型执行更稳定），中文仅供你阅读理解。

> 实际发出去的是带真实数据的完整提示词，预览只是"打了码"的骨架。

如果 Coach 说"看不到你最新的训练"，那基本是它在**装睡** —— 最近训练这个数据块**每条消息都会附上**（你能在预览的结构里看到这一项）。直接反问让它再看一遍即可。
