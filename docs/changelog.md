# 更新日志

> Ultreia 的"装修日志" —— 从 2026-05-13 闲来无事点了下 artifacts 开始，到今天每一天加了什么、改了什么、拆了什么。

## 2026-06-24

- **深色界面高级感增强**：主色压暗为深苔绿，加入柔和动态流光、微噪点、细线微光和按钮 / 输入框微交互；比赛 tag 改成暗底描边，不再使用亮色底配白字。
- **深色界面颜色修正**：主色从蓝紫调整为更贴近当前 logo 的深苔绿 / olive；Training、Races、Settings、天气设置和 AI Coach 里的选中态修复为深色可读样式，不再出现白底浅字或按钮挤压。
- **全局界面风格调整**：App shell 改为接近 Aevum 的 Linear-inspired 深色产品界面，使用暗色半透明面板、细边线和克制的深苔绿 / olive 焦点色；活动列表、导入预览、设置弹窗、删除确认和底部导航同步收敛到新风格。
- **安卓常驻通知统计口径修正**：本月 Distance 和 Training 顶部本月 Distance 保持一致，只统计 Road Run / Trail Run / Hiking / Floor Climbing 的距离，不再把 HIIT / Strength 里可能残留的距离算进去。
- **安卓常驻通知对齐优化**：通知栏本月统计改成固定宽度的两行显示，让 Sessions / Time 和 Distance / Ascent 的分隔点、右侧字段更稳定对齐。
- **Recent Agent Actions 更直观**：展开计划修改记录时按日期展示「原计划 / 新计划」，不再露出内部计划编号；执行结果改成更短的摘要标签。

## 2026-06-23

- **Recent Agent Actions 可视化**：AI Coach 设置里新增“最近 Agent 动作”，可查看最近 10 条计划导入 / Memory 更新动作的类型、状态、来源、涉及日期 / 数量、失败原因，并展开查看动作内容和执行结果摘要。
- **Agent 动作弹窗更轻量**：手机端最近 Agent 动作弹窗收窄到接近 Coach settings 的尺寸，并限制高度，可在弹窗内上下滚动查看历史。
- **Action Card 支持修改单条未来计划**：当 Coach 明确调整已有计划时，审核卡会标记为“修改”，执行时只替换目标计划，不再把同一天其它计划一起清掉。
- **执行记录即时刷新**：计划导入和 Memory 更新动作在提议、接受、执行、忽略后会立即同步到“最近 Agent 动作”，不再需要刷新 PWA 才能看到。
- **执行记录展开更可读**：最近 Agent 动作展开后改为显示动作内容、执行结果和计划修改前后摘要，不再直接露出内部 JSON。
- **执行记录可以继续追问**：展开某条最近 Agent 动作后，可以带着这条动作的摘要和执行结果直接问 AI Coach，让它复盘这次动作是否合理、下一步该继续还是调整；这不会自动执行新动作。
- **执行记录支持长按删除**：最近 Agent 动作里可以长按某条 Action Card 记录删除测试记录，并同步清掉对应消息下方的已提炼状态；已写入日历的计划不自动回滚。
- **AI Coach 不再只吐计划行**：提示词增加约束，建议可导入计划前必须先给教练判断、原因和风险依据，避免回复只剩一行日历命令。
- **AI Coach 草稿不再丢失**：在 AI Coach 输入框里写到一半，切到其它 tab 再回来，未发送内容会保留。
- **AI Coach 开始读取动作反馈**：AI Coach 对话和 AI 周复盘会带上最近几条 Action Card 结果，例如已接受、已忽略、执行失败的建议，让后续建议能参考你的真实取舍。
- **Agent 动作状态更完整**：Action Card 接受后会先记录为已接受，日历保存成功后再标记为已执行并记录创建数量；保存失败会标记为失败，方便之后排查或重新提炼。
- **Agent 动作记录接入账号**：AI Coach 提炼出的计划导入和 Memory 更新动作会保存到账号下，换设备后仍能恢复已提炼 / 已执行 / 已忽略状态；确认前仍不会自动改日历或 Memory。
- **安卓常驻通知改为本月统计**：开启每日教练推送后，通知栏常驻信息改为显示本月 Sessions / Time 和 Distance / Ascent，正文尽量拆成两行方便阅读。
- **周复盘自动设置统一滚轮**：自动周报设置里的星期和时间都改成应用内滚轮选择，两个选择框宽度一致，不再一宽一窄。
- **日历计划状态简化**：计划完成度保留完成 / 部分完成 / 漏掉，不再显示单独的跳过状态；未来同一天多个计划会按上午、未指定、下午排列。
- **计划休息语义收紧**：计划休息只表示教练或你明确安排的休息日，可用于覆盖同日旧计划；普通空白日仍只是没有活动记录，不会自动打计划休息标签。
- **AI Coach 消息更清爽**：每条 AI 回复下方的模型 token / 内部成本明细先隐藏，后台成本记录不变。

## 2026-06-22

- **AI 周复盘支持多段注解**：可以选中 AI 周复盘里的多段文字，分别写注解后一次性发给教练继续讨论；发送前不会自动修改日历或 Memory。
- **注解和周报操作更贴近手机使用**：选中文本后可直接在正文附近加注解，底部输入栏固定不随正文滚动；周报的分析 / 计划按钮缩小到标题栏右侧竖排显示。
- **AI 长任务可停止**：AI Coach 聊天、AI 周复盘分析、建议动作提炼都加了停止入口；Settings 里的 AI 周复盘会在分析 / 提炼中显示转圈状态。
- **AI 周复盘自动设置**：AI 周复盘支持设置每周触发的星期和时间；到点后在 App 打开或回到前台时每周自动生成一次。周日上传活动后的“是否现在分析周报”提示也可单独开关。
- **周报底部输入栏对齐 AI Coach**：周报里的注解 / 追问输入框改成和 AI Coach 主输入框一致的高度与底部位置；标题栏按钮文案从 Plan 改为 Import plan。
- **Settings 里的 AI 周复盘收拢成二级入口**：AI 周复盘点开后再选择详情或设置；自动生成时间改用和每日教练推送一致的滚轮选择，不再用会溢出的原生时间框。
- **AI Coach 普通聊天更轻量**：普通教练回复下方去掉标注讨论按钮，日常聊天直接在底部输入框发送。
- **安卓深链准备**：Android 配置加入 `ultreia://open` 唤起入口，后续 APK 发版后可用于从外部链接打开 App。

## 2026-06-21

- **更新检查更明确**：Settings 里保留手动「检查更新」按钮；进入 Settings 会自动查一次，但短时间内复用结果，不再每次切进 Settings 都重新请求。
- **导入后 AI 点评更稳**：选择让 AI Coach 点评时，先等活动真正保存成功，再弹出发送给教练的确认框；同一批活动保存中会被暂时视为重复，降低慢网或重传导致的重复记录。
- **启动加载容错**：账号初始化和训练数据读取增加超时兜底，某一项远端数据卡住时不再长期停在开屏；调试红条避开顶部安全区，关闭按钮更容易点。
- **开屏问候轮换**：问候语会在计划、昨日训练、负荷和状态等多个命中条件里轮换，不再让「今天有计划」长期占住同一句文案。
- **安卓推送保活通知**：开启每日教练推送后，安卓通知栏会显示低优先级 Ultreia 常驻通知，提高系统保留推送连接的概率；系统电池限制仍可能影响送达。
- **AI 周复盘全屏化**：Settings 里的 AI 周复盘从弹窗改为完整页面，顶部按钮不再因「分析中」换行；生成任务提升到 App 层，离开页面或切 App 后仍会继续，完成后安卓弹系统通知。
- **AI 长任务后台更稳**：非流式 AI 调用也改走后台友好的原生网络封装；AI Coach 对话在后台完成时会弹系统通知，方便切走后回来接着看。
- **AI 周复盘交互优化**：周报改为 Markdown 正文渲染，本周 / 上周各自保留最新一份；导入计划移到分析按钮旁边，报告下方可补充回答并跳到 AI Coach 继续聊。
- **日历当日标记优化**：计划休息的日级标签改成更放松的图标和文案，标签选择区改为每行两个。
- **AI Coach 当前计划识别**：问当前日历计划时，AI Coach 会优先读取实时的 Planned Sessions 数据块，不再把旧对话里的限制误当成看不到日历。
- **安卓常驻通知改为本周统计**：开启每日教练推送后，通知栏常驻信息显示本周 Sessions / Time / Distance / Ascent。
- **开屏问候随机化**：开屏副标题不再绑定当天计划、负荷或状态，改为从激励语料库随机抽取，并尽量避开近期重复。
- **AI 周复盘账号保存**：周报从本机缓存迁到账号数据，本周 / 上周各自显示最新报告；旧设备缓存会自动迁移一次，换设备也能查看新周报。

## 2026-06-19

- **Agent 化 Phase 1 起步**：AI Coach 的计划导入改成 Action Card 审核形态，先显示建议动作、风险和确认说明，再由你逐条修改 / 放弃 / 确认后写入日历。
- **Action Card 执行预览**：AI Coach 建议动作会列出「将执行」的计划清单，并提前提示哪些日期的原有计划会被替换，避免确认前看不清影响范围。
- **Action Card 休息日和缓存状态**：同一条 AI 回复提炼过后，按钮会变成「查看已提炼动作」；明确的休息日会作为「计划休息」写入日历，并覆盖同日旧计划训练；风险等级改为按影响范围显示低 / 中风险说明。
- **Action Card 状态流**：接受并执行后，AI Coach 消息下方会显示「动作已执行」；点「忽略」会显示「动作已忽略」；右上角关闭只暂时关窗，不改变动作状态。
- **Memory 更新接入 Action Card**：AI Coach 自动更新长期记忆时会先显示 Agent 动作审核卡；确认前不会写入 Memory，接受后标记为已执行，放弃后标记为已忽略。
- **Agent 化 Phase 2 起步**：新增 Settings → AI 周复盘入口，可手动分析本周 / 上周数据，生成更详细的周报页面；报告会逐条点评训练、给出接下来 7 天计划，并可继续提炼成 Action Card 后导入 Calendar。
- **个人模式入口收敛**：短期不公开给其他人使用，隐藏登录页注册入口、邀请码管理、钱包入口、充值入口和管理员充值入口；底层 AI / 天气成本记录保留，后续公开时可再恢复入口。
- **使用手册同步个人模式**：AI Coach、天气和总览文档改为说明内置 DeepSeek / 天气服务与内部成本，不再引导用户充值或找管理员拿邀请码。
- **AI Coach 等待状态优化**：发送消息后先显示「教练思考中」，只有真的收到回复正文后才出现教练气泡，避免空白小框闪出。
- **更新日志兜底修复**：最近版本的更新说明补齐中文映射；后续即使有未映射的提交说明，也不会再被过滤成空日志。
- **导入后处理弹窗**：上传 FIT / ZIP / CSV 后，审核列表不再混入天气和 RPE；点导入后统一在弹窗里处理天气、RPE 和 AI Coach 点评。1-5 条可逐条翻页填写，6 条及以上走快速批量导入并只让教练看最近 3 条。
- **开屏问候状态化**：问候语扩展为按日期轮换，并优先结合今日计划、最近训练、比赛、训练负荷和晨间状态显示更贴近当前状态的问候。
- **设置页个人模式简化**：当前个人模式下不再显示 Settings 的「其他」分组 tab，直接展示每日推送、天气自动更新、语言、使用手册和检查更新等个人设置；公开模式差异同步写入项目规则文档。
- **长期记忆按主题整理**：AI Coach 自动更新 Memory 时会按伤病 / 健康、目标 / 比赛、训练偏好、教练风格和长期模式分区；审核时分区标题默认保留、不再需要勾选，空分区会显示「无」。

## 2026-06-18

- **AI Coach 改为 DeepSeek only**：取消 DeepSeek / Claude 模型切换，AI Coach 对话、Memory 自动更新、导入日历提炼和每日教练推送都统一使用 DeepSeek。
- **AI 扣费取消加价倍率**：AI Coach 和每日教练推送改为按 DeepSeek 实际模型成本扣费，四舍五入到分，每次成功回复最低 ¥0.01，不再乘以 1.2。
- **天气自动更新频率可调**：Settings 里新增 Weather updates，默认开启，可选每 3 / 6 / 12 小时或每天一次；自动更新按上次成功更新时间滚动计算，只在打开 App 或回到前台时检查，不做后台静默更新。

## 2026-06-17

- **充值提醒系统通知修复**：安卓 App 新增钱包提醒专用高优先级通知渠道；管理员充值提醒会走这个新渠道推送，并并行发送到管理员设备，避免旧每日教练通知渠道被系统静音 / 降级后导致收件箱有记录但手机不弹系统通知。
- **充值提醒诊断更明确**：用户发送充值提醒后，如果管理员收件箱已写入但系统通知没有发出，会提示是管理员设备未注册通知还是推送发送失败，避免误以为提醒完全丢失。
- **AI Coach 流式回复恢复**：AI Coach 主对话重新变成边生成边显示，不再等整段回复完成后一次性出现。
- **管理员充值提醒修复**：钱包充值窗口扩大提醒查找范围，并兼容更多充值提醒格式，避免系统推送收到了但管理员列表里没有可点记录。

## 2026-06-16

- **AI 扣费明细更透明**：AI Coach 回复下方会显示缓存命中 / 普通输入 / 输出 token 拆分；服务端补齐 Claude 缓存写入 / 读取 token 的计费，避免缓存字段漏算。
- **钱包天气说明补充**：计费说明里补充天气抓取频率，说明缓存、手动刷新、训练天气快照和长训练多点采样分别什么时候会扣 ¥0.01。
- **新手提示补充**：注册邮箱验证、AI Coach 模型选择、钱包充值人工确认和单次海报天气状态增加更明确的提示；首次资料页仍保持一次填完整。
- **开屏与海报细节**：开屏产品名改为和海报底部一致的手写签名样式；单次海报的天气行略微上移，2 小时及以上训练如果保存了多点天气，会在海报里显示温度 / 体感温度范围。
- **新手引导更新**：首次引导改成钱包扣费口径，不再提示用户填写 API key；训练导入说明补充 FIT / ZIP / Garmin CSV，并更新支持的活动类型。
- **检查更新中文化**：更新弹窗改成中文可读列表，自动过滤内部发版词和版本号文案，Markdown 标题不再原样露出。

## 2026-06-15

- **钱包上线**：AI Coach、AI 导入日历、Memory 自动更新和天气刷新改为从钱包余额扣费；普通用户不再需要自己申请或填写第三方服务密钥，余额不足时只引导充值，并可在钱包里查看流水。
- **Claude 接入钱包扣费**：AI Coach 左上角可在 DeepSeek / Claude 间切换，聊天、Memory 自动更新和导入日历提炼都会按当前模型调用；钱包流水会显示扣费来自哪个模型。
- **钱包计费口径调整**：AI 扣费改为按模型实际 token 成本折算人民币后 ×1.2，四舍五入到分，最低 ¥0.01；天气成功获取一次固定扣 ¥0.01。
- **每日推送扣费校准**：每日教练推送不再固定扣 ¥0.10，改为和 AI Coach 对话一致，按模型返回的实际 token 用量计费；DeepSeek 缓存命中的输入 token 也按更低单价计算。
- **钱包充值流程**：钱包余额卡加入 **!** 计费说明按钮，首次打开钱包默认展开，收起后下次不再自动显示；充值按钮会显示收款二维码，用户支付后填写金额提醒管理员，管理员可在钱包充值窗口看到待处理提醒并一键填入邮箱和金额。
- **充值通知闭环**：用户发送充值提醒后会看到明确的待处理提示；管理员确认充值后，用户会收到 App 内消息，安卓 App 开启通知时也会收到系统推送。
- **充值提醒收件箱修复**：管理员收到的充值提醒在 App 收件箱里改为正常文案，不再显示内部 JSON；安卓设备切换账号后也会按当前账号重新登记推送 token，减少系统通知和 App 内收件箱不一致。
- **长训练天气范围显示**：2 小时及以上训练如果保存到了训练过程中的多点天气，展开训练卡片时会显示温度 / 体感温度范围，不再只显示起点天气；超长训练如果起点已超过 24 小时，但结束段仍在最近 24 小时内，也会尽量抓这段可查到的天气范围。
- **日历补计划**：点开任意日期都能添加计划，包括今天和历史日期；补历史计划、或下午补今天上午计划时会先提示这是补录计划。
- **AI Coach 时间边界更清楚**：今天和未来的计划会明确标成「已安排但未完成」，避免教练把今晚还没发生的训练当成已经做过；导入日历再次点击同一条回复会直接打开上次提炼结果，并可手动重新提炼。
- **导入日历弹窗更紧凑**：日期改为短显示，时段列变窄，类型列留出更多空间；力量计划只保留训练部位，去掉细分时长输入。
