# 数据导入（FIT / CSV）

可以上传文件批量/快速导入活动，省去手动录入。支持两种格式：

- **`.fit`** —— 佳明（Garmin）、高驰（Coros）以及绝大多数运动手表**原生导出**的格式，数据最全。一次一个活动。
- **`.csv`** —— 佳明 Connect 网页版的批量导出，一次可导多条。

> **高驰用户注意**：高驰只能导出 `.fit`（没有 CSV 导出），用 `.fit` 上传即可。佳明两种都支持——想一次导很多历史活动用 CSV，想要最全的数据（含心率区间分布、路线）用 `.fit`。

## FIT 比 CSV 多了什么

CSV 只有一次活动的汇总（平均/最大心率、总距离等）。FIT 是手表的原始记录，所以导入 `.fit` 时我们还会额外算出并保存：

- **各心率区间停留时间（Z1–Z5）** —— 图表里的「心率区间时长分布」会用到，AI 教练也能据此判断你的「轻松跑」是不是真的轻松。
- **降采样的 GPS 路线** —— 为后续的成绩分享海报准备（暂不在 app 内画图）。

（CSV / 手动记录的活动没有这两项，不影响其它功能。）

> **几个 FIT 的小注意**：
> - **心率区间分布**需要心率阈值。你资料里填了静息 / 最大心率最准；没填的话会按**年龄估算**(约 220−年龄)兜底，仍能出图。
> - **步频(SPM)**已按双脚换算(FIT 里存的是单脚转速)。

## 从 Garmin Connect 导出 CSV

1. 进 Garmin Connect → **Activities** → 列表视图。
2. 按需要筛选日期或类型（导入器不挑，每行都会让你预览）。
3. 点列表右上角的 **Export CSV** 按钮。
4. 保存到本地。

> 手机也能用这个流程：手机浏览器登录 Garmin Connect 网页版导出 CSV，存到手机，再回到 Ultreia 上传。

## .fit 文件怎么来

- **佳明**：Garmin Connect 网页版打开某次活动 → 右上角齿轮 / ⋯ → **Export Original**（导出的就是 .fit）。手机 App 也能分享/导出原始文件。
- **高驰**：高驰 App 里打开活动 → 分享 / 导出 → 选 **FIT**（或 .fit / 原始文件）。

## 导入流程

1. 打开 Training，进入 Activities 子视图。
2. 点最左边的**上传图标**（`.fit` / `.zip` / `.csv` 都行）：
   - `.fit` —— 单次活动。
   - `.zip` —— **批量**：里面放多个 `.fit`，一次性解析全部(适合一堆手表导出的文件一起导)。文件多时会显示**解析进度条**。
   - `.csv` —— 佳明 Connect 批量导出。
3. 可能依次出现两个审核弹窗：
   - **未知活动类型** —— 识别不了的类型会**按原始名称分组**列出（每组显示有多少条），你为**每一组**选一个目标类型，或选「跳过（不导入）」。批量导入一整个文件夹时，同一种类型只需要选一次，不用一条条点。
   - **重复警告** —— 如果某行的日期/类型/时长跟已有记录完全一致，会让你选 **Skip duplicates（跳过重复）** 或 **Add anyway（强制加入）**。
4. 接着是 **Review** 面板，列出每行的解析结果和勾选框：
   - 不想导入的行取消勾选。
   - 每行第一行是日期 / 类型 / （Road Run 的）配速子类下拉，第二行是具体数据。
   - Road Run 行可以通过下拉框覆盖自动分配的配速子类。
5. 点 **Import** 后，会出现 **完成导入**弹窗：
   - 如果导入的活动里有**最近 24 小时内**的户外 / 天气相关活动，或有超长训练与最近 24 小时重叠，可以选择**获取天气**，用 FIT 文件里自带的 GPS 位置抓天气。Strength 默认不抓天气；完全早于 24 小时窗口的活动抓不到（气象服务只保留过去 24 小时），所以批量导入历史记录时不会抓也不显示这个开关。
   - RPE 必填：1–5 条活动可以逐条填写；6 条及以上填写一个批量 RPE，先统一写入，保存后仍可在训练卡片里逐条修改。
   - Strength 必须选择训练部位：1–5 条活动逐条选择；6 条及以上如果包含 Strength，会用一个批量部位先统一写入，保存后仍可逐条改。
   - 可以选择是否让 AI Coach 点评，并在文本框里写这次的主观感受。开启点评时，应用会先保存活动；保存成功后才弹出发送给教练的确认框，确认前不会消耗 AI 对话额度。聊天里会显示你的主观感受，后台会同时带上新增活动、当前计划和负荷信息，让教练在点评训练的同时给下一次相关计划一个统一口径。你的原始主观感受会由应用本地压缩成短自评备注写回对应训练，不再让教练代写这条备注；教练回复完成后，应用会再用 AI 压缩出一条短教练点评写回同一条训练。训练列表和日历详情里会分成「自评 / Self review」和「教练点评 / Coach review」两栏，每栏最多显示两行。
   - 如果当天是周日，上传保存成功后会额外询问是否现在生成 AI 周复盘；如果同一次导入也勾选了 AI Coach 点评，会先处理点评确认，再出现周复盘提示。点稍后不会自动分析。
6. 点 **Save import** 写入数据库。保存过程中会出现“上载中”提示并暂时阻止其它点击，同一批活动也会被暂时视为重复，避免网络慢或重传时写出两份相同记录。

## 读了哪些字段

会从 Garmin CSV 里找以下列：

| Garmin 列名 | 用途 |
|---|---|
| Activity Type | 决定活动类型 |
| Date | 日期 |
| Distance | 距离（km）|
| Time / Total Time / Moving Time / Elapsed Time | 时长（按顺序找第一个存在的列）|
| Avg HR | 平均心率 |
| Max HR | 最大心率 |
| Total Ascent | 累计爬升 |
| Avg Run Cadence | 平均步频 |

配速自动算出来：`时长 ÷ 距离`（两个都 > 0 时）。

## 活动类型怎么映射

读 Garmin 的 "Activity Type" 列，按关键字判断：

| 类型名包含 | 映射到 |
|---|---|
| trail | Trail Run |
| hiking、walking、walk | Hiking |
| stair、stepper、step machine、floor | Floor Climbing |
| cycling、bike、biking | Cycling |
| swim | Swimming |
| hiit、interval training、crossfit | HIIT |
| cardio | HIIT |
| strength、weight | Strength |
| yoga、pilates、stretch | Strength |
| run（且不含上述）| Road Run |
| 其他 | 标记为未知（让你选）|

> 高驰把健身房综合训练记成「cardio」，所以默认归到 **HIIT**（大多数是这种）；偶尔几条偏力量的，导入后在列表里手动改成 Strength 即可。空类型（手表没记运动项目）会进未知组，由你选 Strength 或跳过。

## 重复检测规则

满足下面**全部**条件视为重复：

- 日期相同
- 类型相同
- 时长（秒）相同

**Skip duplicates** 只跳过重复行；**Add anyway** 全部加入（会出现两条相同记录 —— 只在 Garmin 把一次训练拆成两次导出时才有用）。

## 提示

- CSV 完全在浏览器本地解析。不会传到任何其他地方，只有你确认后的勾选行才入库。
- Garmin 的导出列名会随 app 版本和语言变化。如果导入器提示「找不到时长列」，多半是 Garmin 改了列名 —— 反馈给开发者修。
- 还没支持的活动类型（比如划船）会被识别成「未知」需要手动选 —— 这是有意的保守策略，避免错误归类。
