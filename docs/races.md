# 赛事管理

Races tab 管两件相关的事：你**计划参加**的未来赛事，和你**已经跑过**的历史赛事。它们共用同一个表单。**PR 栏**会从历史里自动聚合每个类别的最佳成绩。

## 两端布局区别

- **电脑**：PR 栏在顶部，下面纵向叠放 Target 区块和 History 区块。
- **手机**：顶部只保留 **Target / History** 两个 tab，标签上带计数；可以点标签，也可以左右滑动切换，内容会跟随方向滑入。PR 汇总收在 History 顶部的可展开区域，不再单独占一层 tab。
- **手机目标赛事卡**：第一行显示优先级、类别、组别和倒计时；第二行从具体日期开始，后面接赛事名称。
- **手机历史赛事卡**：第一行从日期开始，后面接赛事类型、组别和 ITRA 等标签，右侧显示加粗高亮的完赛成绩；第二行只显示赛事名称，长名称会换行；第三行只在 Trail 赛事有距离 / 爬升时显示赛道数据。

下文操作按你当前端找对应入口即可。

## 赛事类别

七个固定类别，按显示顺序：

1. **10K**
2. **Half Mar.（半马）**
3. **Full Mar.（全马）**
4. **Trail（越野）** —— 距离不定，有爬升字段
5. **Spartan（斯巴达）** —— 按 tier 分，下面单列
6. **Hyrox** —— 固定室内赛制，无距离/爬升字段；可选官方组别（男子/女子公开、男子/女子精英、男双/女双/混双、男双 Pro / 女双 Pro、男子/女子/混合接力）
7. **Other（其他）** —— 兜底类别

这个顺序同时决定了表单下拉框和 PR 栏的展示顺序。

### 斯巴达 tier 配色

选 Spartan 类别后会再露出一行 tier 选择，按官方配色显示：

- **Sprint** —— 红
- **Super** —— 蓝
- **Beast** —— 绿
- **Ultra** —— 紫

未选中的 tier chip 左侧有同色细条做参考，选中后整个 chip 填成该色。赛事卡片上的 tier 标签也带上对应颜色，跟你在比赛现场看到的号码布配色一致。难度排序：Ultra > Beast > Super > Sprint。

### Hyrox 组别

选 Hyrox 类别后会露出一行**官方组别**选择：男子公开 / 女子公开、男子精英 / 女子精英、男双 / 女双 / 混双、男双 Pro / 女双 Pro、男子接力 / 女子接力 / 混合接力。选了会显示在赛事卡片上。

如果资料里已填写性别，新增赛事时会默认隐藏另一性别的单人、双人和接力组别；混双和混合接力会保留。编辑旧记录时，原本已经保存的组别仍会显示，方便检查或修正历史数据。

## 目标赛事

按**日期升序**排（最近的下一场在最前）。每条包含：

- **Priority（优先级）** —— A / B / C。A = "决定整个赛季的核心赛"，B = 次要，C = 参与性质。优先级会影响 PR 栏颜色，发给 AI 教练时会写明「Priority A/B/C」让教练知道哪场最重要。
- **类别** + （仅 Spartan）tier
- **日期**
- **距离 / 爬升** —— Trail 和 Other 有；路跑类（10K/HM/全马）距离已由类别决定所以隐藏；Hyrox 也隐藏
- **名称**
- **倒计时** —— 日期还没到的目标赛事，卡片上会显示一个绿色小标，离比赛 7 天以上按"还有 X 周"显示，最后一周内按"还有 X 天"，比赛当天显示"就在今天"。方便你对照训练周期（堆量周 / 减量周）安排。历史赛事和已过期的目标不显示。

### 添加目标赛事

1. Races（手机端：Target tab）→ **Add Target Race**。
2. **先选类别** —— 表单会根据类别动态显示/隐藏字段。
3. 填名称、日期、优先级，按需要填距离/爬升。
4. 按需要填**比赛地点**（见下）。
5. 如果日期填在过去，会弹警告问要不要直接归到历史里。

### 比赛地点 + 比赛日天气

表单里有个可选的**比赛地点**：输入城市名（如"广州"），从弹出的下拉里选一个。精确到城市就够，不用填街道——天气是按区域算的。Hyrox 是室内赛事，不显示这一项。

填了地点的赛事，天气默认**不显示**（列表保持清爽，跟室内赛一样）——**点一下赛事卡**就能展开看当天天气，再点收起。

**目标赛事**展开后显示比赛当天的天气：

- **临近比赛（约两周以内）**：显示当天的**真实预报**（温度区间 + 体感温度）。
- **还早（几个月后）**：显示该地点该日期的**往年同期典型天气**（多年平均的温度区间、体感、大致降水）。几个月后的天气谁也报不准，这里给的是"历年这天大概什么样"，帮你提前判断要不要练热适应、定补给和装备。

**历史赛事**展开后显示比赛**当天的真实天气**（能查到很久以前任意一天，数据滞后约 5 天）：**3 小时以上**的长赛显示当天**温度范围**（最低~最高），更短的显示当天均温。

目标赛事的天气也会进 AI 教练（只带最近一场），详见 [AI 教练](ai-coach.md)。

如果一场目标赛进入 14 天内，且赛事地点 / 天气信息足够明确，AI Coach 会在对话上方自动整理一份赛前简报和装备检查清单。A / B / C 级目标赛都会触发；如果同一天有多场比赛，才按 A > B > C 排优先级。Hyrox 是室内赛，不需要填写地点也可以生成。简报只用于查看和继续追问，不会自动修改日历计划。

## 历史赛事

按**日期降序**排（最近的最前）。表单跟目标一样，多一个**完赛时间**字段（时:分:秒）。Trail 历史还能在 PR 栏上挂一个可选的 **ITRA Performance Index** 标记。

## 筛选

目标和历史各有一行类别筛选。电脑用 chips 多选；手机用下拉单选（一行的 chips 在 360 宽的屏上会换行，下拉更紧凑）。区块标题旁的计数器在有筛选时会显示 `已筛 / 总数`。

## PR 栏

手机端进入 History 后，先点顶部的 **PR** 汇总行展开；电脑端仍固定显示在赛事列表上方。

从历史列表里自动聚合每类的 PR：

- **10K / 半马 / 全马 / Hyrox / Other** —— 按**最快完赛时间**排
- **Trail** —— 按**最长距离**排（越野「PR」更看重耐力而非速度）
- **Spartan** —— 按**最难 tier** 排（Ultra > Beast > Super > Sprint）

每张 PR 卡片下面的「+ other finishes」可以展开看这一类的其他成绩。

### ITRA

Trail 那张 PR 卡的**右下角**有一个 **ITRA** 小标记（跟「+ other finishes」同一行）。点它录入或更新你的全局 ITRA 表现指数。这是**每个用户一个值**，不是每场赛事一个。

## 编辑 / 删除

点任一行进入内联编辑。没分类的行上有个小下拉框，直接设类别不用打开完整表单。卡片右上角的 × 删除（带确认）。

## 跟 AI 教练的关系

- **目标赛事全部**发给教练（教练需要知道你接下来要打哪些比赛）。
- **历史赛事经过筛选**发给教练：每个项目只发**最近一场 + 最好成绩那场**（路跑按最快时间、越野按 ITRA/距离、斯巴达按最难 tier），两个锚点足够，保持提示聚焦。完整规则见 [AI 教练](ai-coach.md)。
