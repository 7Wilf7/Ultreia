# 训练记录

每条训练都归到三大类之一：**跑步 / 耐力（Run / Endurance）**、**力量（Strength）**、**Conditioning（HIIT）**。这份记录是所有统计的源头 —— 顶部统计条、图表、AI 教练能看到的数据、PR 栏，都从这里读。

## 八种活动类型

| 大类 | 具体类型 | 记录哪些字段 |
|---|---|---|
| 跑步 / 耐力 | Road Run（路跑）| 距离、时长、配速、心率、累计爬升、步频 |
| 跑步 / 耐力 | Trail Run（越野跑）| 距离、时长、配速、心率、累计爬升 |
| 跑步 / 耐力 | Hiking（徒步）| 距离、时长、配速、心率、累计爬升 |
| 跑步 / 耐力 | Floor Climbing（爬楼/楼梯机）| 时长、心率、累计爬升（无水平距离）|
| 跑步 / 耐力 | Cycling（骑行）| 距离、时长、**速度（km/h）**、心率、累计爬升 |
| 跑步 / 耐力 | Swimming（游泳）| **距离（米）**、时长、**配速（每 100 米）**、心率 |
| 力量 | Strength | 时长、心率、可选身体部位 |
| Conditioning | HIIT | 时长、心率 |

子类规则因大类而异：

- **Road Run** 必须选一种配速子类（Easy / Aerobic / Tempo / Interval），可加 Race 标记。详细规则见 [Running 配速分类](running.md)。
- **其他跑步类型**（Trail / Hiking / Floor Climbing）可加 Race 标记，但不分配速 —— 地形决定配速，按心率分类会误导。
- **Cycling / Swimming** 没有子类，也不按心率分配速。骑行看**速度（km/h）**，游泳看**每 100 米配速**，跟跑步的配速逻辑不同。游泳距离用**米**录入和显示。它们归在「跑步 / 耐力」组下（铁三三项放一起），但**不**计入「全部跑步 / 耐力」的跑量统计 —— 想单看骑行或游泳，在筛选器里单独选。
- **Strength** 可勾选 Upper Body / Lower Body / Core 任意组合。
- **HIIT** 没有子类。

> **日级标签**不是训练类型，而是在日历上某一天勾选的状态标记：**按摩 / 拉伸 / 睡眠差 / 生病 / 出差**。一天可以勾多个。这些会喂给 AI 教练 —— 生病、出差尤其影响它怎么帮你排计划。在 Calendar 点开某天就能勾选。

> **今日状态（晨间自评）**：在 Calendar 点开今天（或过去某天），底部有「今日状态」——给**睡眠 / 腿感 / 精力**各点一个「差 / 一般 / 好」（再点一下取消）。一秒钟的事，但教练会据此判断你今天该顶还是该收。未来日期不显示（还没发生）。

每条训练还有两个**可选**字段，所有类型都能填：

- **RPE（自觉用力，1–10）** —— 这次训练你感觉有多累，1 = 很轻松，10 = 拼尽全力。这是个主观打分，拿不准就留空。填了之后 AI 教练能更准地估算你的训练负荷。
- **备注** —— 一句话随手记，比如"换了新鞋""膝盖有点紧""昨晚没睡好"。会原样喂给 AI 教练当作背景。

## 手机端卡片显示

手机端每张训练卡分两行。第二行是按类型定制的紧凑指标，**时长统一放最前面**，方便竖向扫读：

| 类型 | 紧凑指标顺序 |
|---|---|
| Road Run | 时长 · 距离 · 配速 |
| Trail Run / Hiking | 时长 · 距离 · 爬升 |
| Floor Climbing | 时长 · 爬升 |
| Cycling | 时长 · 距离 · 速度 |
| Swimming | 时长 · 距离（米）· 每 100 米配速 |
| Strength / HIIT | 时长 · 心率 |

剩下的字段（步频、最大心率…）点开卡片展开后才显示。

## 添加训练（手动）

1. 打开 Training，进入 Activities 子视图（手机和电脑都默认在这）。
2. 点操作行最左的**上传图标**旁边的 **+** 添加图标。
3. 选日期和类型。Road Run 默认子类 Easy Run，其他类型默认空。
4. 按你测到的填。**日期和时长必填**，距离/心率/爬升/步频都可选 —— 没量到就留空。
5. 「开始时间」可选 —— 留空就用保存这一刻的天气；填了就抓那个时间点的历史天气（最近 24 小时）或未来预报。
6. **获取天气**复选框 —— 只有当这条训练在**最近 24 小时内**（或未来）时才会出现：户外类型（路跑/越野/徒步/骑行）默认勾选、室内类型（力量/爬楼/游泳）默认不勾，但**任何类型都能手动勾**（比如你在户外做的力量）。不勾就不调天气，省额度。如果开始时间填的是**超过 24 小时以前**，这个复选框直接不显示 —— 气象服务只保留过去 24 小时的数据，再早的查不到。
7. **RPE** 和**备注**可选填（见上方说明）。填了会让 AI 教练的判断更准。
8. 保存。新行**立刻**出现在列表顶部，表单同时关闭 —— 天气抓取和云端写入都挪到了后台跑，不会再卡屏。如果云端写入失败，刚加的行会消失，并弹一条错误提示让你重试。

> 手机端的跑步子类 chip 用短文案（Easy / Aerobic / Tempo / Interval），节省横向空间；电脑端仍然是完整的 Easy Run / Aerobic Run / Tempo Run / Interval Run。

## 天气

每条手动添加的训练会自动附上当时的天气快照（温度、体感、湿度、天气状况、风速、AQI），显示在卡片上。

- **第一次添加时**会请求位置权限。授权后用手机/电脑的 GPS 定位，精确到街道。
- **拒绝授权 / 没装定位 / 海外 VPN 卡住**的情况：去个人资料 → 位置，点"用当前位置"或手填坐标兜底。
- **上传 FIT 文件导入**时，如果其中有**最近 24 小时内**的活动，审核面板里会出现一个**获取天气**开关（默认开），给这些活动抓天气，用的是 FIT 文件里自带的 GPS 位置（比当前定位更准）。更早的活动抓不到，所以批量导入历史记录时不会抓。
- **CSV 批量导入**和**日历计划**不抓天气（前者是历史数据，后者要看那天才知道）。
- **过去的训练**（开始时间填的是 24 小时内的某一刻）会抓那个时间点的历史天气。
- **未来的训练**（开始时间填的是接下来 7 天内）会抓那天的预报，保存进去。

天气抓不到时（位置拒绝、网络不通、彩云额度用完）静默跳过，记录正常保存，只是少了天气字段，不影响其它功能。

## 编辑训练

- **电脑**：点列表里任一行进入内联编辑。点外面取消 —— 有未保存改动会弹确认框。
- **手机**：点卡片展开 → 点 **Edit** 按钮。

## 批量操作

- Activities 顶部是一行**纯图标**操作按钮：**上传（最左、黑底，主入口）· 添加 · 多选 · 时长筛选 · 排序**。点上传从手表文件导入是现在的主要方式，所以放最前并加重；手动加一条用「添加」图标。
- **上传**：支持 `.fit` / `.zip` / `.csv`，详见 [数据导入](data-import.md)。
- **添加**：手动录入单条训练。
- **多选**：勾选多行批量删除（图标上会显示已选数量）。
- **排序**：按日期、距离、时长或心率排序当前列表。
- **顶部固定** —— 筛选、Activities / Charts 切换、时间范围、统计（SESSIONS/TIME/DISTANCE/ASCENT）、操作按钮和训练负荷条会作为一个整体钉在 Training 页顶部，往下滑只有活动列表滚动。
- **训练负荷条**（操作按钮下方）—— 用最近 **7 天** vs 最近 **4 周** 的「时长 × RPE」算一个比值（ACWR），帮你看出最近加得是不是太猛：圆点绿=理想 / 灰=偏低 / 土黄=偏高 / 橙=骤增（>1.5，伤病高风险）。攒够约 4 周训练才准（不够显示「基线建立中」），RPE 填得越全越准。这条也会发给 AI 教练。
- **下拉刷新** —— 在 Training 页顶部往下拉，会强制重新拉取云端数据（顶部出现加载指示）。数据平时只在 App 启动时拉一次、不会自动同步，所以在别的设备改了之后，回到手机下拉一下即可看到最新。
- **双击回顶部** —— 在 Training 页双击底部导航的「训练」图标，活动列表平滑滚回最顶。

## 筛选和时间范围

- **筛选** —— 顶部 **All activities ▼** 是个**两级菜单**：点开先看到三个大类（跑步 / 耐力、力量、Conditioning），点某个大类，右边一列才列出它的二级分类。可按跑步子类（含 Cycling / Swimming）、力量身体部位、HIIT 筛选。选「全部跑步 / 耐力」只聚合跑步类，不含骑行游泳；想看骑行或游泳要单独选。筛选同时作用于 Activities 列表和图表。
- **时间范围** —— **All Time / This Week / This Month / This Year / Custom** 标签，控制统计条和列表的范围。Custom 可选起止日期。**图表有自己独立的时间范围**，详见 [图表](charts.md)。

## 关于计划训练

从 AI 教练那里一键导入到日历的**计划训练**只在日历上以虚线框显示。它们**不**进 Training 列表，**不**进统计条，**不**进图表，**不**进 PR 栏。只有等你手动标记完成后才会计入。

**过期的计划会自动淡出**——过了当天，月历上那条计划会变浅、不再扎眼，你**不用再手动删**它。点开那天，每条过期计划会显示结果：那天有完成训练就自动算**完成**；没有就标**漏掉**，旁边给你「✓ 已完成 / 跳过」两个一键按钮（主动跳过的不算漏）。AI 教练会看到最近两周的**计划依从**（完成几次、漏几次），漏了会先问你原因再继续排。

> 教练导入计划时会**覆盖**那天已有的计划，而不是叠加——比如原计划 5 公里、教练分析后改成 8 公里，导入后那天只剩 8 公里这一条。

在日历点开某一天，**长按**某条训练记录,会弹出「编辑」「删除」两个按钮(平时藏起来,让列表清爽):

- **编辑**(仅计划训练)—— 改类型、距离、时长、时段（上午 / 下午）；力量训练还能改部位。改完点「保存计划」即可。
- **删除** —— 移除这条记录。

计划用虚线框跟已完成训练区分，所以行里不再单独标「计划」两个字。
