# 天气

应用会自动抓取你所在位置的天气，灌给四个地方：

1. **训练记录**上每条手动添加的活动 —— 卡片紧凑视图行尾、展开后的完整面板
2. **日历下方** —— 7 张未来天气卡（今天 + 接下来 6 天）
3. **日历日期弹窗顶部** —— 该天的天气总结
4. **AI 教练** —— 每次发消息时把实时 + 未来 7 天预报作为上下文一起发给教练

数据源是**彩云天气**（一家国内的气象 API 服务商）。

天气由 Ultreia 服务端代为拉取，不需要你自己申请或填写第三方天气服务密钥；每次成功获取天气从钱包扣 ¥0.01。

## 你能看到的天气数据有哪些

每条天气快照尽量包含：

- **温度**（°C，预报会同时给最高/最低，实时给当前值）
- **体感温度**（feels-like，热应激判断的核心，跟绝对温度可以差 5–8°C）
- **湿度**（%）
- **天气状况**（晴 / 多云 / 阴 / 小雨 / 雷雨 / 雾霾 / 雪 / 大风 / 浮尘 ...）—— UI 里会用对应的小图标表示
- **风速**（km/h，小于 1 不显示）
- **AQI**（中国标准空气质量指数，0 不显示）

某些字段（比如历史快照里的体感）算不出来时会自动跳过，不会渲染成"null"或者"0"。

## 数据什么时候会更新

不同数据有不同的"保质期"，搞清楚每条的节点很重要，否则容易误判"为什么显示的还是上午的温度"。

### 实时天气

| 触发 | 什么时候真正发请求 |
|---|---|
| 打开应用 | 缓存里有不到 **3 小时**的实时数据 → 直接用缓存。超过 3 小时 → 重新抓 |
| 切回 app（从后台 / 切 tab 回来） | 同上：缓存够新就不抓，过期就抓 |
| 日历下方的 **↻ 刷新按钮** | 强制重新抓，无视缓存 |
| 更换默认位置 | 立即重新抓 |

所以：你看到的实时温度**最多比真实情况老 3 小时**。要拿当下数据，按那个 ↻ 按钮即可（在日历页 7 天天气卡片那一栏的右上角）。

### 7 天预报

| 触发 | 什么时候真正发请求 |
|---|---|
| 打开应用 | 缓存里有**当天**的预报 → 直接用。换了一天 → 重新抓 |
| 切回 app | 同上 |
| 日历下方的 ↻ 按钮 | 强制重新抓 |
| 更换默认位置 | 立即重新抓 |

预报**每天最多更新一次**（彩云的日级数据本来就是日级的，更高频也没有更多信息）。新的一天打开应用会自动拉新。中间想强刷也是按 ↻。

### 手动添加训练里的天气

添加活动时填了"开始时间"会决定抓哪种：

- **不填开始时间** → 抓**保存这一刻的实时天气**
- **填了过去 24 小时内的时间** → 抓那个**时间点的历史天气**（气象服务提供过去 24 小时的小时级数据）
- **填了未来 24 小时之后的时间** → 抓**那天的预报**

> **完全早于 24 小时窗口的训练抓不到天气**：气象服务只保留过去 24 小时的逐小时记录，再早的没有。所以当整条训练都在一天以前时，"获取天气"复选框会**直接消失**——不是 bug，是确实没数据可抓。这也避免了把"现在的天气"错误地贴到一条几天前的训练上。

> **超长训练例外**：如果训练本身很长，虽然起点已经超过 24 小时，但结束段仍然落在最近 24 小时内，应用会尽量抓最近 24 小时内那一段的天气范围；更早的起跑段仍然拿不到。

> **上传 FIT 文件**导入时同理：审核面板里只有当存在 24 小时内的活动，或有超长训练与最近 24 小时重叠时，才会出现"获取天气"开关，给这些活动抓天气（用 FIT 自带的 GPS 定位）；完全早于 24 小时窗口的活动会跳过。

抓到之后**直接写进这条训练记录里**，**不会自动更新**。也就是说，一条训练保存下来后，它显示的天气就是"当时记录下的那个"，跟今天实时天气没关系了 —— 这才是对的，因为这条天气是要让 AI 教练评估"你那次跑步时是什么环境"。

如果训练时间达到 **2 小时或以上**，应用会尽量在训练过程中多取几个时间点的天气。训练卡片展开后会显示这次训练期间的温度 / 体感温度范围，AI 教练也会拿到这条变化轨迹。

> 如果保存时位置抓不到（权限拒了 / 没网 / 彩云额度满了），天气会**静默跳过**，训练正常保存，只是少了天气字段。

### 日历上的当日天气

- **今天和未来 6 天** → 用上面说的 7 天预报数据（今天用的是实时）
- **过去的某一天** → 用那天**第一条训练里记录的天气**作回顾参考；那天没记任何训练的话日历弹窗里就没有天气总结

### AI 教练 prompt 里的天气

每次你按发送的瞬间，prompt 里会塞进三块天气：

- **实时天气**（当前时刻）—— 用上面的实时缓存
- **未来 7 天预报**（每天一行）—— 用上面的预报缓存，**无论那一天你有没有训练计划都会发**
- **近 10 条训练每条行尾的当时天气**（记录时抓的那个，不会变）

教练拿到这些数据后会基于天气调整建议，比如"明天最高 33°C 体感 38°C，把那个 tempo 挪到周四（最高 28°C）"。**不刷新天气直接发消息**用的就是缓存里的数据，所以如果你刚强刷了天气再发消息，教练会拿到最新数据；不刷直接发，教练拿到的是缓存数据（最多 3 小时旧）。

## 位置（在「个人资料」里设置）

天气抓取的**第一步**是确定经纬度。优先顺序：

1. **设备 GPS**（浏览器 / APP 内的定位接口）—— 准、就近
2. **个人资料里设置的位置坐标** —— GPS 抓不到时用这个兜底

什么时候 GPS 抓不到：

- 手机 APP 没授予位置权限
- 浏览器拒绝了位置请求
- 在某些限制网络环境下，自动定位会卡住或超时
- 设备根本没装 GPS（旧电脑）

**位置现在统一在「个人资料」里设置**（以前在设置页的"默认位置"已并到这里）。打开个人资料 → 位置：

- 点 **📍 用当前位置**：自动定位 + **自动填好地址**（按 App 语言显示成"广州市天河区"这种区/市级地址）+ 保存坐标。用这个的话**一个字都不用打**。
- 也可以手动输入地区名（给教练看的地址），以及手填经纬度做兜底（**WGS84 国际坐标**，跟 Google Maps 同一套；百度地图自带偏移，不能直接用）。

这一个位置**同时**用于：天气抓取（坐标）和 AI 教练的地理上下文（地址）。换城市/出差**记得更新一下**。

## 我看到的天气好像不准 / 没更新

按下面顺序排查：

1. 看日历下方的 7 天天气卡片有没有显示。**没显示** → 多半是定位拿不到，去个人资料里设置位置
2. **显示了但是温度奇怪**：看右上角的"更新于 HH:MM"，如果是几个小时前的时间，按 **↻ 按钮**强刷一次
3. **强刷后转圈但没结果**：检查钱包余额；余额足够但仍失败时，可能是天气服务临时异常
4. **训练卡片上的天气不对**：那是**记录的时候**抓到的天气，**不会跟着今天的实时天气变**。要改的话只能删了重录

## 哪些活动会显示天气标记

**只要这条训练记录里存了天气，卡片上就会显示天气标记**，跟类型无关。

- 户外类型（路跑/越野/徒步/骑行）添加时默认就抓天气，所以基本都有。
- 室内类型（力量/爬楼/游泳）默认不抓，通常没有；但如果你是在**户外做的力量**、手动勾了"获取天气"，那这条也会照样显示天气标记。

AI 教练拿到的训练记录里，**凡是存了天气的那条就会带上天气**（任何类型都一样）。
