Training Studio · 文件夹导览

这个项目里每个文件夹都是干嘛的

写给看得懂产品、不一定看代码的你。下面把项目拆成几大类，每个文件夹一句人话解释。 带 可删 的是能安全删掉的，带 自动生成 的删了也会自己长回来。

🤔 为什么根目录一堆"散装"文件，没装进文件夹？

因为它们必须待在最外层——这是各种工具的硬规矩，不是乱放。打包工具、依赖管理器、安卓封装器启动时， 第一件事就是去项目根目录找自己那份"说明书"（比如 package.json、vite.config.js、index.html）。 你把它们塞进子文件夹，工具就找不到、整个项目跑不起来。所以这些散文件是正常的，不用动。 真正装"内容"的，都在下面那些文件夹里。

应用本体 云端/服务端 安卓打包 文档 配置文件 工具脚本 可删/自动生成

📱 应用本体 —— App 的"血肉"，绝大多数改动都在这

src/ 75 个文件 核心
整个 App 的源代码就在这。你在手机/网页上看到的每个页面、按钮、弹窗，都是这里的文件画出来的。下面是它的几个抽屉：
components/界面积木（38 个）——每个页面/弹窗一个文件，如训练页、日历页、AI 教练页、各种设置弹窗。改界面基本都在这。
components/Auth/登录、注册、账号徽章这几块单独放。
lib/"工具库"（20 个）——天气、对外请求、FIT 手表文件解析、推送这些幕后能力。
lib/db/和云数据库打交道的一层（13 个）——每张表（训练、赛事、对话…）一个文件，负责读写。
data/写死的小数据（2 个）——开屏问候语、新手教程文案。
i18n/中英文翻译（2 个）——切语言时用到的对照表。
hooks/ · utils/复用的小逻辑和零碎工具函数（登录状态、时间格式、AI 提示词拼装等）。

🎨 图片资源 —— 原样发布的图标和图

public/3 个文件保留
网页用的图标、favicon、开屏 logo。这里的东西会原封不动出现在网站上。
resources/3 个文件保留
安卓 App 的图标和启动图原图（高清大图），打包工具拿它生成各种尺寸。海报功能的 logo 水印也在这。

☁️ 云端 / 服务端 —— 跑在别人服务器上的代码

supabase/6 个函数保留
数据库后台的"云函数"。这些不是在手机上跑，而是在 Supabase 云上跑的小程序：每日推送、邀请码注册、注销账号、免费额度的 AI/天气代理。
⚠ 改了这里的代码，前端发版不会自动带上，要单独手动部署（细节在 CLAUDE.md）。
api/1 个文件保留
Vercel（部署网站那家）上的一个小后端函数——天气代理，让密钥不暴露在浏览器里。

🤖 安卓打包 —— 把网页装进手机 App 的壳

android/61 个文件保留·别手改
安卓 App 工程，由 Capacitor 这个工具自动生成的"壳"，把你的网页包成能装在手机上的 APK。 里面文件多、看着吓人，但基本不用手碰——大多是工具维护的。少数几个原生插件（下载安装包、保存海报到相册）是手写的。

📖 文档 —— 给人看的说明书

docs/10 个文件保留
给最终用户看的中文使用手册（就是那个 GitBook 网站）。每个功能一页，加一份更新日志。这是你刚让我精简的那批。
README.md保留
项目门面简介（打开 GitHub 仓库第一眼看到的）。
CLAUDE.md AGENTS.md保留
给 AI 助手（我）看的协作规则——发版流程、踩过的坑、数据库约定都写在这。两份内容一样，给不同 AI 工具读。
PRODUCT.md保留
产品层面的说明/定位。

⚙️ 配置文件 —— 工具的"说明书"，必须在最外层

package.json package-lock.json命根子
列出 App 用到的所有第三方零件清单，以及锁定每个的精确版本。版本号（你 bump 的那个）也在这。
vite.config.js保留
打包工具的配置——怎么把源代码压成能上线的成品、PWA 设置都在这。
index.html保留
网页的最外层骨架（一个几乎空的壳），App 启动时往里填内容。
capacitor.config.json · eslint.config.js保留
分别是安卓封装配置和代码体检规则（lint）。
.env.local · .env.example别删·别外传
密钥/网址（连数据库用的）。.env.local 是你本机真实的密钥，不上传 git、删了本地就连不上；.env.example 是给别人看的"空模板"。
.gitignore保留
一张"别上传名单"，告诉 git 哪些文件/文件夹不要传到 GitHub（密钥、安装包、自动生成的东西都在名单上）。

🔧 工具脚本 / 自动化 —— 偶尔用一次的辅助

.github/1 个文件保留
发版流水线。你一推 tag，GitHub 就照这里的剧本自动构建、签名、发布安卓 APK。你那条"发 APK"的魔法就靠它。
scripts/4 个文件保留
一次性小工具脚本——造测试数据、生成 App 图标/启动图、检查文件名大小写。平时不跑，需要时手动调。
.claude/保留
Claude Code（我）的本地配置。不影响 App 本身。

🗑️ 自动生成 / 可删 —— 占地方但不珍贵

node_modules/自动生成·最占地方
下载下来的几百个第三方零件，通常几百 MB，是文件夹里最大的一坨。不上传 git。真删了也没事——一句 npm install 就全装回来。
dist/自动生成
打包成品（上线用的压缩版）。每次 build 都会重新生成，删了无所谓。
.git/别删
git 的版本库内部（所有历史记录都在这）。占地方但千万别删——删了就丢掉全部提交历史。
启动.bat你自己的·可留可删
你本地的一个双击启动脚本（跑本地预览）。不上传 git，纯属方便自己。用不上就删，没影响。

这次我顺手清理了什么：

🗑️ 删了 src/data/samples.js（最早做 demo 时写死的假训练数据，全项目已经没人用了）
🗑️ 删了 src/App.css（最初模板自带的样式，从没被引用过）
📦 补装了本机缺的一个依赖（你从公司电脑拉来的新代码需要它，否则本地构建会报错）

结论：这个项目其实不乱——根目录的散文件都是工具规定要放那的，文件夹分得也清楚。真正"冗余可删"的只有上面那两个死文件（已删）和自动生成的 node_modules / dist。
